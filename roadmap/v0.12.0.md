# v0.12.0 — SPARQL Update Advanced

**Effort estimate: 3–4 person-weeks**

**Goal:** SPARQL 1.1 Update defines a rich set of write operations beyond the simple INSERT DATA and DELETE DATA implemented in v0.5.1. This release completes the standard with pattern-based updates, loading from URLs, and full graph management operations (clear, drop, create, copy, move).

---

## Pattern-Based DELETE/INSERT WHERE

**Update triples based on a query pattern — not just fixed values.**

The `DELETE { … } WHERE { … }` and `INSERT { … } WHERE { … }` forms of SPARQL Update let you express transformations: "for every entity that has an old type, replace it with a new type", or "for every measurement above a threshold, add a flag triple". The WHERE clause is a full SPARQL query — any pattern you can SELECT, you can use as the basis for an update. This is the workhorse operation for knowledge graph maintenance and migration scripts.

---

## LOAD from URL

**Fetch and import an RDF document directly from a web URL.**

The `LOAD <url>` operation fetches an RDF document from an HTTP or HTTPS URL and imports its triples into the specified named graph. This is useful for pulling in publicly available linked data (DBpedia, Wikidata snippets, published ontologies) without first downloading and staging the file. The URL allowlist configured via `pg_ripple.federation_endpoints` applies here too, preventing SSRF (server-side request forgery) to internal network addresses.

---

## CLEAR, DROP, and CREATE Graph Operations

**Manage named graph lifecycles with standard operations.**

`CLEAR GRAPH <iri>` removes all triples from a named graph without deleting the graph name itself. `DROP GRAPH <iri>` removes both the triples and the graph metadata. `CREATE GRAPH <iri>` ensures a named graph exists even before any triples are added to it. These operations complete the basic graph administration toolkit needed for data pipelines that stage data in temporary graphs, then promote or discard them.

---

## Full SPARQL 1.1 Update Compliance

**This release closes the gap to full specification coverage.**

Combined with the operations from v0.5.1 and the earlier SPARQL query engine, this release reaches compliance with the SPARQL 1.1 Update specification for the core operations. The W3C conformance test suite for Update operations is added to the CI pipeline. The remaining Update operations (MOVE, COPY, ADD) are implemented in v0.48.0 alongside other standards-completeness work.
