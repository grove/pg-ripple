# v0.68.0 - Distributed Scalability, Streaming Completion and Fuzz Hardening

> **Full technical details:** [v0.68.0-full.md](v0.68.0-full.md)

**Status: Released ✅** | **Scope: Large**

> v0.68.0 completes the distributed-execution and streaming contracts that were partially delivered in v0.62–v0.66 but not fully wired, introduces nonblocking VP promotion, and adds scheduled fuzzing for all twelve fuzz targets.

---

## What is this?

After v0.67.0 closes the Critical security and correctness findings, v0.68.0 makes the distributed and streaming headline features real rather than labelled stubs: Citus HLL aggregate translation, Citus SERVICE and multihop pruning in the SPARQL translator, true streaming CONSTRUCT iterators, nonblocking VP promotion with shadow tables and crash recovery, and a fuzz CI schedule for all twelve fuzz targets.

## Headline outcomes

- SPARQL `GROUP BY COUNT(DISTINCT …)` routes through Citus HLL when `pg_ripple.approx_distinct` is on; exact fallback when HLL extension is absent.
- Citus SERVICE endpoint pruning and multihop cost reduction are wired into the SPARQL translator, gated behind explain and correctness tests.
- CONSTRUCT Turtle/JSON-LD cursor helpers stream triples from a database portal without materializing the full result set.
- Nonblocking VP promotion uses a shadow table pattern: background copy, atomic rename, RLS propagation, progress state in `_pg_ripple.predicates`, and crash recovery.
- Scheduled CI workflow runs all 12 fuzz targets nightly for a short duration; a longer manual workflow supports extended coverage campaigns.
