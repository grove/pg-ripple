# v0.82.0 — Assessment 12 Performance & Observability

> **Full technical details:** [v0.82.0-full.md](v0.82.0-full.md)

**Status: Planned** | **Scope: Large**

> v0.82.0 addresses the thirteen performance findings and seven observability findings from Assessment 12, together with the remaining security hardening items. Key wins: a configurable plan cache, `ANY($1)` binding in batch decode, two-phase merge with shorter fence lock, an enriched EXPLAIN output that includes the SPARQL algebra tree, a merge worker heartbeat, structured Prometheus labels, and a `sparql_max_algebra_depth` GUC for DoS protection.

---

## What is this?

With correctness and security locked down in v0.80.0–v0.81.0, v0.82.0 focuses on the performance and observability layer. The changes make the system faster under production workloads, easier to diagnose with `EXPLAIN`, and more secure against resource-exhaustion attacks.

## Headline outcomes

- [x] **CACHE-CAP-01** — Add `pg_ripple.plan_cache_capacity` GUC (integer, default 1024, min 64, max 65536) replacing the hardcoded `1024` constant in `src/sparql/plan_cache.rs`.
- [x] **DECODE-BIND-01** — Migrate `batch_decode()` in `src/dictionary/mod.rs` from `IN (id1, id2, …)` to `WHERE id = ANY($1::bigint[])` bind parameter to prevent plan proliferation.
- [x] **MERGE-PRED-01** — Cache predicate ID lookups in the merge worker to avoid repeated `_pg_ripple.predicates` scans per cycle.
- [x] **FED-COST-01** — Materialise a `_pg_ripple.federation_stats` table with endpoint latency P50/P95 and row-count estimates; use it in federation cost-model decisions.
- [x] **STATS-CACHE-01** — Add `_pg_ripple.predicate_stats_cache` materialised table, updated by a background refresh, so `graph_stats()` does not perform per-query VP-table scans under load.
- [x] **ARROW-LIMIT-01** — Add `pg_ripple.arrow_max_export_rows` GUC (default 10,000,000) and enforce it in the Arrow Flight export endpoint, returning HTTP 400 with a clear error when the limit is exceeded.
- [x] **PGSS-NORM-01** — Normalise SPARQL query text (replace literals and IRIs with `$N` placeholders) before passing to `pg_stat_statements`, so equivalent parametric queries share a single stats row.
- [x] **GUC-BOUNDS-01** — Add explicit `min_val` / `max_val` validators to `vp_promotion_threshold`, `dictionary_cache_size`, and `merge_batch_size` GUC definitions.
- [x] **METRICS-LABELS-01** — Add `query_type` (SELECT/ASK/CONSTRUCT/DESCRIBE/UPDATE) and `result_size_bucket` labels to the `pg_ripple_http_sparql_query_duration_seconds` Prometheus histogram.
- [x] **EXPLAIN-ALG-01** — Add `"algebra"` key to `sparql_explain()` JSON output showing the SPARQL algebra tree (after optimisation) as a nested JSON object.
- [x] **MERGE-HBEAT-01** — Emit a `LOG`-level heartbeat in the merge background worker every `pg_ripple.merge_heartbeat_interval_seconds` seconds (default: 60), including last-merge timestamp, predicate count, and row count processed.
- [x] **STATS-DOC-01** — Document `graph_stats()` scan behaviour in the SQL reference and add `pg_ripple.stats_scan_limit` GUC to cap the number of VP tables scanned per call.
- [x] **REDACT-01** — Audit and unify `redacted_error()` usage across all `pg_ripple_http` handlers; ensure stack traces never reach HTTP response bodies.
- [x] **SPARQL-COMPLEX-01** — Add `pg_ripple.sparql_max_algebra_depth` GUC (default: 64); return HTTP 400 with `PT429` code when a query's algebra depth exceeds the limit.
- [x] **TENANT-NAME-01** — Validate `tenant_name` at the SQL API boundary using a regex allowlist (`^[A-Za-z0-9_]{1,63}$`) before constructing trigger names.
- [x] **ROLE-UNICODE-01** — Add an SPI fallback path for `quote_ident_safe()` when the role name contains Unicode characters that the Rust `pg_quote_identifier()` function does not handle.
- [x] **FED-SIZE-01** — Cap the federation HTTP response body at `pg_ripple.federation_result_max_bytes` (default: 10 MiB) using `ureq`'s `limit()` reader, not a post-read truncation.
- [x] **SHMEM-SAFE-01** — Replace direct multiplication in shared-memory size calculations (`src/lib.rs`) with `checked_mul().expect("shmem overflow")` (the expect is justified: overflow means misconfigured GUC).
- [x] **RUSTSEC-01** — Review all `[advisories.ignore]` entries in `audit.toml`; remove any that are no longer relevant; add new exemptions for RUSTSEC-2023-0071 (RSA/PKCS#1 timing) if no fix is available.
- [x] **ADMIN-LOCK-01** — Document the lock levels acquired by `ripple_vacuum()`, `ripple_reindex()`, and `recover_stuck_promotions()` in the administration reference.
