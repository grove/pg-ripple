# v0.48.0 — SHACL Core Completeness, OWL 2 RL Closure & SPARQL Completeness

**Effort estimate: 9–11 person-weeks**

**Goal:** This release completes three feature sets simultaneously. The remaining seven SHACL Core constraints are implemented, finishing the SHACL standard. Five missing OWL 2 RL inference rules are added, completing the rule set. Several SPARQL standard features — MOVE/COPY/ADD graph operations, SPARQL-star variable-in-quoted-triple patterns, and federation response limits — that were partially or not implemented are now fully wired in.

---

## Remaining Seven SHACL Core Value Range Constraints

**`sh:minLength`, `sh:maxLength`, `sh:xone`, and the four exclusive/inclusive range constraints.**

`sh:minLength` and `sh:maxLength` validate that a string literal's length falls within bounds — essential for data quality rules like "account numbers must be exactly 12 characters." `sh:xone` (exclusive or) validates that a node conforms to exactly one of a list of shapes, not zero and not two or more. `sh:minExclusive`, `sh:maxExclusive`, `sh:minInclusive`, and `sh:maxInclusive` apply numeric range constraints to numeric literal values. With these implemented, all SHACL Core constraint operators are available.

---

## Complex `sh:path` Expressions

**Nested path expressions — inverse, alternative, sequence, and Kleene closures — all work.**

SHACL paths follow the same syntax as SPARQL property paths. Simple paths (a single predicate) were already supported. This release implements the full path expression syntax in SHACL: `sh:inversePath` (follow a predicate in reverse), `sh:alternativePath` (follow either of two predicates), sequence paths (follow two predicates in sequence, expressed as an RDF list), and the Kleene-star, plus, and optional operators (`*`, `+`, `?`). Complex shapes like "the manager of the manager of the manager of this person must have the right to authorize this budget" are now expressible.

---

## Five Completing OWL 2 RL Rules

**The missing rules for property chain axioms, reflexive properties, and asymmetric properties.**

The OWL RL rule set was previously almost complete but missing five rules that are required for conformance: the `owl:propertyChainAxiom` rule (if `p` is declared a chain of `q` followed by `r`, and `A q B r C`, then `A p C`); the `owl:ReflexiveProperty` rule; the `owl:IrreflexiveProperty` constraint; the `owl:AsymmetricProperty` constraint; and the `owl:HasSelf` rule. With all five added, the OWL 2 RL rule set is complete and the W3C OWL 2 RL conformance suite target rises to ≥95%.

---

## SPARQL Update: MOVE, COPY, ADD

**The three graph-to-graph copy operations complete the SPARQL Update specification.**

`COPY source TO dest` copies all triples from the source graph to the destination, leaving the source intact. `MOVE source TO dest` copies then clears the source. `ADD source TO dest` merges the source into the destination without clearing either. These three operations were defined in the SPARQL 1.1 Update specification but not implemented. They are now compiled to atomic SQL transactions.

---

## SPARQL-star Variable Inside Quoted Triple

**`SELECT ?who WHERE { <<?who ex:likes ex:Coffee>> ex:certainty ?c }` now returns results.**

SPARQL-star allows querying quoted triples where one of the quoted triple's terms is a variable. For example, finding all subjects whose liking-of-coffee has a high certainty requires a variable in the subject position of the quoted triple. This pattern was parsed successfully but failed to produce results because the variable was not correctly bound during SQL generation. The fix propagates variable bindings through the quoted-triple path in the pattern translator.

---

## Federation Response Size Limit

**Runaway federation calls cannot fill the pg_ripple server's memory.**

A new `pg_ripple.federation_max_response_bytes` GUC (default 100 MB) limits how much data is accepted from any single federation endpoint in a single request. If an endpoint streams more bytes than the limit, the request is terminated with an error. This protects pg_ripple against misbehaving or malicious federation endpoints that stream unbounded data.
