# v0.61.0 ‚Äî Ecosystem Depth & Polish ‚Äî Full Details

**Theme:** Explainability, GDPR, standards completeness, dbt/Kafka ecosystem
integrations, SHACL-AF execution, observability depth.
**Scope:** Large  
**Prerequisite:** v0.60.0

---

## Why this release

v0.60.0 closes the v1.0.0 blockers. v0.61.0 widens the surface for enterprise
adopters: compliance tooling (GDPR erasure, per-graph access control, audit
explainability), standards completeness (SPARQL Entailment Regimes, SHACL-AF
rule execution, OWL 2 RL deletion proof), and ecosystem integrations (dbt
adapter, Kafka CDC sink) that make pg_ripple a first-class citizen of the modern
data stack.

---

## Deliverables

### Access control & compliance

- **[6.3 / L-8.1] Per-named-graph row-level security** ‚Äî Apply PostgreSQL RLS
  policies to every VP table and `vp_rare`, filtering by
  `g IN (SELECT graph_id FROM _pg_ripple.tenant_graphs WHERE tenant_role = current_user)`.
  Extend `pg_ripple.create_tenant()` (v0.57.0) to install the corresponding
  RLS policy; add `pg_ripple.grant_graph(graph_iri, role)` and
  `pg_ripple.revoke_graph(graph_iri, role)` helper functions.
- **[6.7 / L-8.3] GDPR right-to-erasure** ‚Äî New SRF
  `pg_ripple.erase_subject(iri TEXT) ‚Üí TABLE(relation TEXT, rows_deleted BIGINT)`.
  Removes all triples about the subject across all VP tables, `vp_rare`, the
  dictionary (if unreferenced), the KGE embeddings table, the PROV-O named
  graph, and the audit log. Runs in a single transaction; returns a per-relation
  deletion count for the erasure record.

### Inference explainability

- **[6.6] `pg_ripple.explain_inference(s, p, o, g)`** ‚Äî New SRF returning the
  rule-firing provenance chain for a given inferred triple as a JSON derivation
  tree: `(depth INT, rule_id TEXT, source_sids BIGINT[], child_triples JSONB[])`.
  Implemented by walking the `source` column (explicit = 0, inferred = 1) and
  the Datalog rule-firing log introduced in this release.

### SPARQL standards completeness

- **[6.8 / B7-2 / B-1] SPARQL Entailment Regimes test driver** ‚Äî New
  `tests/sparql_entailment/` directory running the W3C SPARQL 1.1 Entailment
  Regime test suite against pg_ripple's combined SPARQL + Datalog stack.
  Covers RDFS Entailment and OWL 2 RL Entailment sub-suites. Added to CI as
  `entailment-suite` (informational first release, blocking in v1.0.0).

### SHACL completeness

- **[D7-1 / D-3 / S4-8] SHACL-AF `sh:rule` execution** ‚Äî Implement the bridge
  in `src/shacl/af_rules.rs` that compiles `sh:rule` triple patterns to Datalog
  rules and loads them via `load_rules_text()`. Add a pg_regress test
  `tests/pg_regress/sql/shacl_af_rule_execution.sql` that loads a SHACL-AF
  shapes graph with `sh:rule` and asserts inferred triples appear.
- **[D7-2] `sh:closed` + `sh:ignoredProperties`** ‚Äî Add pg_regress test
  covering the corner case where `sh:ignoredProperties` contains a property
  also used on the focus node by another property shape.

### Datalog correctness

- **[6.13 / E7-2] OWL 2 RL deletion proof test** ‚Äî New
  `tests/pg_regress/sql/datalog_owl_rl_deletion.sql` that inserts 100k synthetic
  triples, runs OWL 2 RL inference to materialization, then deletes all base
  facts and asserts that zero inferred triples remain. Exercises the full
  DRed retraction path.
- **[E7-2] DRed cycle guard pg_regress fixture** ‚Äî Add
  `tests/pg_regress/sql/datalog_dred_cycle.sql` constructing a `sameAs` cycle
  and asserting PT530 is raised.

### Ecosystem integrations

- **[6.11] dbt adapter** ‚Äî Publish `dbt-pg-ripple` package in
  `clients/dbt-pg-ripple/` with SPARQL macros (`sparql_model`, `sparql_source`,
  `sparql_ref`); include a sample dbt project.
- **[6.12 / L-6.2] Kafka CDC sink** ‚Äî New module `src/kafka.rs` (behind
  `feature = "kafka"`) implementing a Kafka producer wired to the
  `pg_ripple_cdc_lifecycle` NOTIFY channel. Configurable via GUC
  `pg_ripple.kafka_brokers` and `pg_ripple.kafka_topic`. Requires `rdkafka`
  crate.

### Observability

- **[I7-1] OTLP traceparent propagation** ‚Äî Extract `traceparent` header in
  axum middleware (`pg_ripple_http`); pass via session-local GUC
  `pg_ripple.tracing_traceparent`; propagate into the extension's tracing context
  before each SPARQL/Datalog query span. Enables end-to-end distributed traces
  from load balancer through HTTP service into the extension.
- **[I7-2] OpenTelemetry semantic-convention map** ‚Äî New doc
  `docs/src/operations/observability-otel.md` with a span-name ‚Üí attribute
  table and example Prometheus / Grafana queries for pg_ripple spans.
- **[I7-3] Federation call stats per endpoint** ‚Äî Extend
  `pg_ripple.federation_call_stats()` to return
  `(endpoint TEXT, calls INT, errors INT, blocked INT, p50_ms INT, p95_ms INT, last_error_at TIMESTAMPTZ)`.
- **[F7-3] BRIN summarize visibility** ‚Äî Promote persistent
  `brin_summarize_new_values` failures from `debug1` to `NOTICE` after the second
  consecutive merge cycle where the call fails; add `brin_summarize_failures INT`
  column to `_pg_ripple.predicates`.

## Exit criteria

- `shacl_af_rule_execution.sql` pg_regress test passes end-to-end.
- `erase_subject()` removes all traces of a subject in a single transaction.
- `explain_inference()` returns a well-formed JSON tree for at least one
  OWL RL-derived triple.
- `entailment-suite` CI job is added (informational).
- `dbt-pg-ripple` package passes its own `dbt test` suite.
- Migration script `sql/pg_ripple--0.60.0--0.61.0.sql` present.
- `pg_ripple.control` updated to `default_version = '0.61.0'`.
