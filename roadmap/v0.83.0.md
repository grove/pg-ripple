# v0.83.0 — Assessment 12 Test Coverage, API Polish & Code Quality

> **Full technical details:** [v0.83.0-full.md](v0.83.0-full.md)

**Status: Planned** | **Scope: Large**

> v0.83.0 completes the Assessment 12 remediation track. The focus is depth of test coverage (8 new error-path tests, 3 new fuzz targets, stronger proptest invariants, async CDC test patterns), API consistency (JSON-LD rename, GUC naming convention, BREAKING changelog tags), supply chain maintenance (serde_cbor evaluation, Renovate config), and code quality (bidi.rs module split, replication.rs panic elimination already in v0.81.0, unwrap audit). After this release the Assessment 12 coverage map is complete and the project advances to the v1.0.0 production hardening track.

---

## What is this?

This is the final remediation release in the Assessment 12 track. It closes the test-coverage tier (T-02 through T-08), the API tier (A-02 through A-06), and the supply-chain tier (DS-02, DS-03) from Assessment 12, together with the remaining code-quality items (Q-01, Q-03, Q-04).

## Headline outcomes

- [x] **ERRPATH-01** — Add regression tests for 8 error-path scenarios: dictionary overflow, HTAP merge during DROP, SubXact abort during encode, federation timeout, Arrow export row limit exceeded, SPARQL depth limit exceeded, tenant-name validation failure, CDC slot exhaustion.
- [x] **FUZZ-BULK-01** — Add fuzz targets for N-Triples, N-Quads, and TriG bulk loaders (`src/bulk/ntriples.rs`, `nquads.rs`, `trig.rs`). Add to the weekly scheduled fuzz CI workflow.
- [x] **FUZZ-UPDATE-01** — Add fuzz target for the SPARQL Update executor (`src/sparql/execute.rs` `sparql_update()` path). Seed corpus from existing `tests/sparql/` UPDATE test files.
- [x] **PROPTEST-02** — Strengthen 5 proptest suites by adding reference-implementation comparisons: `sparql_query()` vs `oxigraph` on SELECT patterns; `datalog_query()` vs a Prolog interpreter on simple Datalog programs.
- [x] **CDC-ASYNC-01** — Replace all `std::thread::sleep(Duration::from_secs(N))` assertions in CDC integration tests with `LISTEN pg_ripple_cdc_ready` / `NOTIFY`-based synchronisation barriers.
- [x] **KFAIL-DOC-01** — Add `# Reason:` comments to every entry in `tests/expected/known_failures.txt` explaining why the test is expected to fail and which GitHub issue tracks the fix.
- [x] **REG-TESTS-01** — Add regression tests for the 13 pg_extern functions identified in Assessment 12 Appendix C as untested: `ripple_vacuum`, `ripple_reindex`, `sparql_cursor_next`, `sparql_cursor_close`, `import_turtle_url`, `export_ntriples`, `export_jsonld_framed`, `load_jsonld`, `cdc_subscribe`, `cdc_unsubscribe`, `federation_endpoint_add`, `federation_endpoint_remove`, `datalog_assert`.
- [x] **API-RENAME-01** — Add `load_jsonld(url TEXT, graph_uri TEXT DEFAULT NULL)` SQL alias and deprecate `json_ld_load` with a `NOTICE` message; remove `json_ld_load` in v1.0.0.
- [x] **API-GRAPH-COL-01** — Add `g BIGINT` column to `RETURNS TABLE` definitions of VP-row–returning functions so named-graph membership is accessible without a separate join.
- [x] **GUC-NAME-01** — Document the pg_ripple GUC naming convention (`pg_ripple.noun_verb_unit` snake_case) in `CONTRIBUTING.md`. Identify the 4 GUCs that do not follow the convention and add deprecation notices.
- [x] **CHANGELOG-BREAK-01** — Adopt `BREAKING:` tag convention in `CHANGELOG.md` for user-visible API removals or incompatible GUC changes. Back-annotate the 3 breaking changes in v0.73.0–v0.79.0 that were not tagged.
- [x] **DEPAUDIT-01** — Evaluate replacement of `serde_cbor` (unmaintained) with `ciborium` or removal if the only usage is the Arrow IPC dictionary batch path. Document the decision and update `Cargo.toml`.
- [x] **RENOVATE-01** — Add a `renovate.json` configuration that auto-creates PRs for Rust toolchain updates in `rust-toolchain.toml` and direct dependency bumps in `Cargo.toml` on a weekly schedule.
- [x] **MOD-BIDI-01** — Split `src/bidi.rs` (2,509 lines) into four focused sub-modules: `src/bidi/relay.rs`, `src/bidi/subscribe.rs`, `src/bidi/sync.rs`, and `src/bidi/protocol.rs`. Re-export the public API from `src/bidi/mod.rs`.
- [x] **P-05-EVAL** — Investigation spike: evaluate moving the dictionary decode LRU cache from process-local memory to PostgreSQL shared memory (`PgSharedMem`). Produce a design doc or close with a "not worth it" rationale.
- [x] **BUILD-TIME-FIELD-01** — Rename the `"build_time"` field in the `/health` JSON response (`pg_ripple_http/src/routing/admin_handlers.rs`) to `"build_version"` or populate it with an actual RFC-3339 build timestamp from `build.rs`. Currently it contains a Cargo version string, not a time.
- [x] **DL-COST-GUC-01** — Replace the hardcoded selectivity divisors `100`, `10`, `1` in `src/datalog/compiler.rs` cost-based rule reordering with GUCs: `pg_ripple.datalog_cost_bound_s_divisor` (default 100), `pg_ripple.datalog_cost_bound_so_divisor` (default 10).
- [x] **CHANGELOG-FMT-01** — Fix the one `CHANGELOG.md` entry missing the bracket-wrapped version format (`## 0.X.Y` → `## [0.X.Y]`). Add a CI lint (`grep -Pn '^## [^[]' CHANGELOG.md`) that rejects entries with incorrect heading syntax.
