# v0.67.0 - Assessment 9 Remediation, Production Evidence and World-Class Hardening

> **Full technical details:** [v0.67.0-full.md](v0.67.0-full.md)

**Status: Released ✅** | **Scope: Very Large**

> v0.67.0 resolves all four Critical findings and eleven High-priority findings from PLAN_OVERALL_ASSESSMENT_9 — storage mutation contract, tenant RLS coverage, Arrow Flight security/correctness, and release-truth gates — then gathers the production-readiness evidence needed to make v1.0.0 credible.

---

## What is this?

Assessment 9 identified four Critical execution-path failures that must be closed before any production-evidence sprint is meaningful: CONSTRUCT writeback bypasses SPARQL Update and bulk load; graph RLS protects only `_pg_ripple.vp_rare`; Arrow Flight accepts unsigned tickets and exports tombstoned rows; and release-truth scripts can exit 0 while checking nothing. v0.67.0 fixes all four, adds the high-priority hardening items that follow from them, and then turns the corrected system into repeatable release artifacts: soak tests, public benchmarks, upgrade/backup/restore acceptance, and threat-model closure.

## Headline outcomes

### Critical remediation
- Storage-level graph mutation journal wired to every insert, delete, batch, SPARQL Update, and bulk-load path.
- CONSTRUCT writeback, provenance cleanup, and CWB metrics driven by the mutation journal, not selected wrappers.
- Graph RLS applied at VP table creation, promotion, grant/revoke, and migration for every predicate.
- Arrow Flight requires a signed ticket in production mode; unsigned tickets are a development-only flag.
- Arrow export reads storage with tombstone exclusion and streams record batches via paged cursor.
- Release-truth scripts reimplemented in portable Python/Rust, fail-closed, version-argument required.
- `feature_status()` evidence paths validated in CI; stale references are a build failure.
- SBOM regenerated on every release and version-checked against `Cargo.toml`.
- Benchmark workflow fixed; benchmark parse failures are a CI failure.
- `construct_rules.rs` panic replaced with a recoverable PostgreSQL error.

### Production evidence
- 72-hour minimum and 30-day target soak-test plan with published artifacts.
- Third-party security audit or tracked internal threat model closure.
- Public BSBM, WatDiv, LUBM, bulk-load, merge, vector, Arrow export, and Citus benchmark baselines.
- Upgrade, pg_dump/restore, backup/restore, and rollback acceptance tests.
- Release evidence dashboard becomes a required artifact for v1.0.0.
