# v0.87.0 — Uncertain Knowledge Engine

> **Full technical details:** [v0.87.0-full.md](v0.87.0-full.md)

**Status: Planned** | **Scope: Very Large**

> v0.87.0 introduces an uncertain knowledge engine built on four composing features: probabilistic Datalog with `@weight` rule annotations and noisy-OR multi-path combination, fuzzy SPARQL filtering via `pg:fuzzy_match()` and `pg:token_set_ratio()`, confidence-threshold property paths (`pg:confPath()`), soft SHACL scoring with `sh:severityWeight` annotations, and automatic provenance-weighted confidence derived from existing PROV-O source trust metadata. All features share a single `_pg_ripple.confidence` side table keyed on the statement ID, leaving the VP table hot path unaffected when probabilistic mode is off. Confidence rows are lifecycle-managed: retraction cascades delete orphaned rows, the HTAP merge worker cleans tombstoned SIDs in-transaction, and a `vacuum_confidence()` helper handles any residual orphans.

---

## What is this?

Classical knowledge graph systems reason in binary: a fact is true or false, a constraint passes or fails. Real-world knowledge is graded — extracted facts carry extraction confidence, sensor readings carry measurement uncertainty, and data quality is more informative as a percentage than a boolean.

v0.84.0 closes that gap by adding a graded confidence layer on top of the existing VP storage, SPARQL engine, Datalog evaluator, and SHACL validator without replacing any of them.

## Headline outcomes

- [ ] **CONF-TABLE-01** — New `_pg_ripple.confidence (statement_id BIGINT, confidence FLOAT8, model TEXT, asserted_at TIMESTAMPTZ, PRIMARY KEY (statement_id, model))` side table created by the v0.84.0 migration script. GIN index on `statement_id` for efficient per-statement lookups. All confidence-bearing features share this table.
- [ ] **PROB-DATALOG-01** — `@weight(FLOAT)` annotation syntax added to the Datalog rule parser. Semi-naive evaluation generates SQL that multiplies body-atom confidences by the rule weight (conjunction) and applies noisy-OR across independent derivation paths. Controlled by `pg_ripple.probabilistic_datalog` GUC (default off). Acyclic rule sets are evaluated exactly; cyclic sets require `pg_ripple.prob_datalog_cyclic = on` to opt in to approximate evaluation.
- [ ] **SPARQL-CONF-FN-01** — `pg:confidence(?s, ?p, ?o)` SPARQL function compiles to a JOIN against `_pg_ripple.confidence`, returning the highest-confidence value across all models or `0.0` if absent. Usable in `FILTER`, `BIND`, and `ORDER BY`.
- [ ] **LOAD-CONF-01** — `pg_ripple.load_triples_with_confidence(data TEXT, confidence FLOAT8 DEFAULT 1.0, format TEXT DEFAULT 'ntriples', graph_uri TEXT DEFAULT NULL) RETURNS BIGINT` bulk-loader that assigns a uniform confidence score to all inserted triples.
- [ ] **FUZZY-SPARQL-01** — `pg:fuzzy_match(a, b)` and `pg:token_set_ratio(a, b)` SPARQL functions compile to PostgreSQL `similarity()` and `word_similarity()` from `pg_trgm`. The v0.84.0 migration script creates a GIN trigram index on `_pg_ripple.dictionary (value gin_trgm_ops)` if `pg_trgm` is available. `pg_ripple.default_fuzzy_threshold` GUC (FLOAT8, default 0.7) is used when no explicit threshold is provided.
- [ ] **FUZZY-PATH-01** — `pg:confPath(predicate, min_confidence)` property-path operator compiles to a `WITH RECURSIVE … CYCLE` CTE that joins against `_pg_ripple.confidence` in the recursive edge condition, traversing only edges whose confidence is at or above the threshold.
- [ ] **SOFT-SHACL-01** — SHACL property shapes may declare `sh:severityWeight xsd:decimal`. New `pg_ripple.shacl_score(graph_iri TEXT) RETURNS FLOAT8` function returns a weighted quality score in [0, 1]. New `pg_ripple.shacl_report_scored(graph_iri TEXT) RETURNS TABLE (focus_node TEXT, shape_iri TEXT, result_severity TEXT, result_severity_score FLOAT8, message TEXT)` function includes per-validation-result scores.
- [ ] **PROV-CONF-01** — `pg:sourceTrust FLOAT8` predicate registered in the VP catalog. Built-in Datalog rule template propagates source trust to `_pg_ripple.confidence` when `pg_ripple.prov_confidence = on` (bool, default off). Noisy-OR combines independent source attestations automatically.
- [ ] **PROB-CI-01** — CI regression suite with adapted ProbLog CLUTRR micro-benchmark and Nations benchmark: assert that derived confidences are within 0.01 of hand-computed reference values for acyclic rule sets.
- [ ] **CONF-GC-01** — Confidence row lifecycle and garbage collection: retraction cascade deletes confidence rows when triples are deleted via `DELETE DATA`, SPARQL Update, or `retract_inferred()`; HTAP merge worker cleans tombstoned SIDs inside the merge transaction; `pg_ripple.vacuum_confidence()` helper purges any residual orphans; `pg_dump` round-trip includes confidence rows.
- [ ] **CONF-HTAP-01** — HTAP merge durability: tombstoned SID cleanup runs inside the merge transaction so confidence reads never see stale scores on the new main partition. SIDs for surviving triples are globally immutable and require no migration.
- [ ] **CONF-EXPORT-01** — Confidence serialization in RDF export: `export_turtle_with_confidence()` emits RDF* reified confidence annotations; JSON-LD 1.1 `@annotation` blocks added when `pg_ripple.export_confidence = on` (default off); N-Quads fallback representation documented.
- [ ] **CONF-PERF-01** — Performance regression suite: `benchmarks/probabilistic_overhead.sql` asserts < 5% hot-path overhead when GUC is on but queries do not call `pg:confidence()`; `benchmarks/confidence_join_scale.sql` asserts < 1 ms median latency for `pg:confidence()` lookups over 100k triples.
- [ ] **CONF-DOCS-01** — Documentation and operator guide: `docs/src/features/uncertain-knowledge.md` chapter; six new GUC entries in configuration reference; upgrade-path note in migration script; `feature_status()` rows for all four sub-features.
