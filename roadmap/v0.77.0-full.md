## Deliverables

### TOOLCHAIN-PIN-01 — Pin rust-toolchain.toml to specific stable version

**Problem**: `rust-toolchain.toml` pins `channel = "stable"` instead of a specific version. Builds are not reproducible across CI runner updates.

**Deliverable**:
1. Pin to a specific version (e.g. `channel = "1.87.0"`).
2. Configure Dependabot or Renovate to update the pin weekly.

**Affected files**: `rust-toolchain.toml`, `.github/dependabot.yml` or `renovate.json`.

### RLS-HASH-01 — Upgrade RLS policy hash from 64-bit to 128-bit

**Problem**: Policy name at `src/security_api.rs:160` uses 64-bit xxh3 hash for graph-scoped policy names. At 4 billion graphs, collision probability is ~50%.

**Deliverable**:
1. Switch to XXH3-128 for policy name generation.
2. Migration script to rename existing policies (or accept coexistence since names are internal).

**Affected files**: `src/security_api.rs`, migration script.

### ARROW-PIN-01 — Pin arrow dependency to minor version

**Problem**: `pg_ripple_http/Cargo.toml` specifies `arrow = "55"` — semver allows any 55.x, which may include breaking changes in practice.

**Deliverable**:
1. Pin to specific minor: `arrow = "55.1"` (or current latest 55.x).

**Affected files**: `pg_ripple_http/Cargo.toml`.

2. Document findings. If any paths leak secrets, add a `RegisterEmitLogHook`.

**Affected files**: Survey across `src/`, document in `docs/src/operations/security.md`.
