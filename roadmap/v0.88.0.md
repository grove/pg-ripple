# v0.88.0 — Datalog-Native PageRank

> **Full technical details:** [v0.88.0-full.md](v0.88.0-full.md)

**Status: Planned** | **Scope: Large**

> v0.88.0 implements iterative PageRank entirely inside pg_ripple's Datalog engine, leveraging aggregation in rule bodies (Datalog^agg), subsumptive tabling for convergence-aware early termination, and magic-sets transformation for goal-directed partial-graph evaluation. A `pg:pagerank()` SPARQL function surfaces ranked nodes inline; `pg_ripple.pagerank_run()` provides a SQL entry point with full parameter control; a materialized `pagerank_scores` VP-aligned view with a BRIN index enables fast top-N lookups; `pg:topN()` aggregates ranked windows directly in SPARQL SELECT; and a **pg-trickle incremental refresh path** propagates rank deltas using bounded K-hop local push so that edge insertions and deletions update scores in milliseconds without a full recompute.

---

## What is this?

PageRank assigns a numerical importance score to every node in a directed graph based on the structure of incoming links. In an RDF triplestore the graph is implicit: any predicate whose values are IRIs defines edges. v0.88.0 makes PageRank a first-class feature by expressing the iterative computation as Datalog rules and routing it through pg_ripple's existing optimizer — magic sets, tabling, parallel stratum evaluation, and semi-naive evaluation all apply automatically.

Full recomputes are practical for batch workloads, but a single new edge should not force re-ranking the entire graph. v0.88.0 integrates with pg-trickle's Z-set differential dataflow to propagate rank deltas locally: when a VP delta table fires, a pg-trickle stream trigger enqueues the changed edge, and a background IVM worker applies bounded K-hop rank propagation (decaying at rate $d^k$, terminating when the delta falls below the convergence threshold). The `pagerank_scores` table gains `stale` and `stale_since` columns so callers can distinguish exact from approximated scores.

## Headline outcomes

- [ ] **PR-DATALOG-01** — Built-in Datalog rule set for PageRank: `pg_ripple.pagerank_rules` GUC selects the active edge predicate(s) (comma-separated IRI list, default `''` meaning all object-valued predicates). Rules emit `pg:hasPageRank` triples on convergence. Controlled by `pg_ripple.pagerank_enabled` GUC (bool, default off).
- [ ] **PR-ITER-01** — Convergence-aware early termination: `pg_ripple.pagerank_max_iterations` GUC (INT, default 100), `pg_ripple.pagerank_convergence_delta` GUC (FLOAT8, default 0.0001). Tabling detects a fixed point and halts without exhausting the iteration cap. `explain_inference()` reports `iterations`, `converged`, and `final_delta` for PageRank strata.
- [ ] **PR-DAMPING-01** — Configurable damping factor via `pg_ripple.pagerank_damping` GUC (FLOAT8, default 0.85). Dangling-node redistribution redistributes rank mass from nodes with zero out-degree uniformly across all nodes in the same component (controlled by `pg_ripple.pagerank_dangling_policy` GUC: `'redistribute'` | `'ignore'`, default `'redistribute'`).
- [ ] **PR-BLANK-01** — Blank-node exclusion policy: `pg_ripple.pagerank_include_blank_nodes` GUC (bool, default false). When off, blank nodes are removed from the edge set before computation and excluded from result triples.
- [ ] **PR-PERSONAL-01** — Personalized PageRank: `pg_ripple.pagerank_run(seed_iris TEXT[], bias FLOAT8 DEFAULT 0.15, ...)` overload accepts a seed-node array and replaces the uniform teleportation vector with a bias concentrated on the seed set.
- [ ] **PR-SPARQL-FN-01** — `pg:pagerank(?node)` SPARQL function compiles to a lookup against the materialized `pagerank_scores` view (or triggers an on-demand run if the view is stale and `pg_ripple.pagerank_on_demand = on`). Usable in `FILTER`, `BIND`, `ORDER BY`, and `HAVING`.
- [ ] **PR-TOPN-01** — `pg:topN(?score, N)` aggregate function returns the top N nodes by descending score as a JSON array. Compiles to `ORDER BY … LIMIT` with BRIN-index range scan on `pagerank_scores`.
- [ ] **PR-SQL-FN-01** — `pg_ripple.pagerank_run(edge_predicates TEXT[] DEFAULT NULL, damping FLOAT8 DEFAULT 0.85, max_iterations INT DEFAULT 100, convergence_delta FLOAT8 DEFAULT 0.0001, graph_uri TEXT DEFAULT NULL) RETURNS TABLE (node_iri TEXT, score FLOAT8, iterations INT, converged BOOL)` SQL function. Writes results to the `pagerank_scores` VP-aligned view transactionally.
- [ ] **PR-VIEW-01** — `pagerank_scores (node BIGINT, score FLOAT8, computed_at TIMESTAMPTZ, iterations INT, converged BOOL, stale BOOL NOT NULL DEFAULT false, stale_since TIMESTAMPTZ)` table in `_pg_ripple` schema, populated by `pagerank_run()` (exact) or the pg-trickle IVM worker (incremental). BRIN index on `score DESC` for top-N range scans. `stale = true` marks nodes whose scores were updated by K-hop local push rather than a full recompute. Exposed as a SPARQL-queryable named graph `pg:pagerank` via a synthetic VP entry. `pg:pagerank(?node)` returns the stored score regardless of staleness; callers may add `FILTER(!pg:isStale(?node))` to restrict to exact scores.
- [ ] **PR-MAGIC-01** — Magic-sets integration: when `pg:pagerank(?node)` appears in a query with a bound `?node`, the Datalog optimizer applies demand transformation to restrict the PageRank computation to the subgraph reachable from or to the seed node. `explain_sparql()` reports `pagerank_scope: partial` vs `pagerank_scope: full`.
- [ ] **PR-TRICKLE-01** — pg-trickle incremental refresh: when `pg_ripple.pagerank_incremental = on` (bool, default off; requires pg-trickle ≥ 0.5) and `pg_ripple.trickle_integration = on`, each VP delta-table insert/delete fires a pg-trickle Z-set stream trigger that enqueues `(source, target, +1/-1)` into `_pg_ripple.pagerank_dirty_edges`. A background IVM worker drains the queue, applies bounded K-hop rank propagation (up to `pg_ripple.pagerank_khop_limit` hops, default 30; terminates early when the delta magnitude falls below `pagerank_convergence_delta`), writes updated scores to `pagerank_scores`, and sets `stale = true` / `stale_since = now()` for affected rows. A scheduled full `pagerank_run()` (configurable via `pg_ripple.pagerank_refresh_schedule` cron expression, default `'0 3 * * *'`) resets `stale = false`. `pg_ripple.vacuum_pagerank_dirty()` helper purges processed entries. Graceful degradation: if pg-trickle is absent, `PR-TRICKLE-01` functions return `PT0408` and the rest of PageRank is unaffected.
- [ ] **PR-HTTP-01** — pg_ripple_http REST API: `POST /pagerank/run` (triggers full computation, returns job ID), `GET /pagerank/results` (top-N results, paginated; `?exact_only=true` filters out stale rows), `GET /pagerank/status` (last run metadata: `computed_at`, `converged`, `stale_count`), `POST /pagerank/vacuum-dirty` (drains the dirty-edges queue). All endpoints authenticated and documented in OpenAPI spec.
- [ ] **PR-CI-01** — pg_regress suite: PageRank on the Karate Club graph (34 nodes, 78 edges) asserts top-3 nodes match NetworkX reference output within 1e-4; convergence within 30 iterations; personalized PR seed-bias regression; `pg:pagerank()` SPARQL FILTER and ORDER BY; `pg:topN()` aggregate; incremental refresh regression (add/remove one edge, assert updated score within 1e-3 of full recompute within 1 hop for local edges, `stale` flag set and cleared correctly).
- [ ] **PR-BENCH-01** — `benchmarks/pagerank.sql` benchmark: 1M-edge synthetic scale-free graph; asserts < 60 s wall time for 30 iterations; BRIN top-100 lookup < 5 ms.
- [ ] **PR-DOCS-01** — `docs/src/features/pagerank.md` chapter covering algorithm, GUCs, SQL API, SPARQL API, REST API, performance tuning, incremental refresh design (K-hop local push, staleness model, full-recompute schedule), and worked examples (social network ranking, entity importance in a knowledge graph, incremental live-graph scenario). `feature_status()` rows for all PR sub-features.
- [ ] **PR-EXPLAIN-01** — `explain_datalog()` PageRank plan output: iteration count, convergence delta per step, dangling-node redistribution annotation, magic-sets scope annotation.
- [ ] **PR-ERR-01** — Error catalog: PT0401–PT0408 covering invalid damping factor (PT0401), invalid edge predicate IRI (PT0402), on-demand run with stale view and GUC off (PT0403), `pg:topN()` non-positive N (PT0404), `pg:pagerank()` called outside a SELECT context (PT0405), max-iterations exceeded without convergence (PT0406), `pg:isStale()` called on non-pagerank node (PT0407), and incremental refresh requested without pg-trickle installed (PT0408).
- [ ] **PR-MIGRATE-01** — `sql/pg_ripple--0.87.0--0.88.0.sql` migration script: creates `_pg_ripple.pagerank_scores` table (including `stale` and `stale_since` columns), `_pg_ripple.pagerank_dirty_edges` stream table, BRIN index on `score DESC`, synthetic VP entry for `pg:pagerank` named graph, new GUC registrations (including `pagerank_incremental`, `pagerank_khop_limit`, `pagerank_refresh_schedule`), and `feature_status()` rows. Rollback section documents manual reversal steps.
