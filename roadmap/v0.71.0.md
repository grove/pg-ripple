# v0.71.0 - Arrow Flight Validation, Citus Integration Tests, and Compatibility Hardening

> **Full technical details:** [v0.71.0-full.md](v0.71.0-full.md)

**Status: Released ✅** | **Scope: Large**

> v0.71.0 closes the High-severity gaps from Assessment 10 that require runtime infrastructure beyond static analysis: confirming Arrow Flight truly streams without buffering the full result, proving Citus RLS propagation end-to-end on a multi-node cluster, and documenting the extension / HTTP companion compatibility contract.

---

## What is this?

Assessment 10 identified three High gaps that could not be closed by code inspection alone: the Arrow Flight `Body::from_stream` vs buffered-body question, the Citus integration test that `feature_status()` claims but does not exist, and the absence of a compatibility matrix between `pg_ripple` and `pg_ripple_http` versioning. v0.71.0 resolves all three with concrete tests and documentation.

## Headline outcomes

- Arrow Flight endpoint confirmed to use `Body::from_stream`; an HTTP-level integration test exports 10 million triples and asserts process RSS stays below a documented bound.
- `tests/integration/citus_rls_propagation.sh` implemented using `docker-compose.yml`; `feature_status()` citation verified.
- Citus HLL accuracy bounds documented in `docs/src/reference/approximate-aggregates.md`; accuracy regression test added.
- `pg_ripple_http` / `pg_ripple` compatibility matrix added to `docs/src/operations/compatibility.md`; HTTP companion startup check rejects incompatible extension version.
- Citus SERVICE shard annotation benchmarked: `EXPLAIN` shows pruned shard count with and without `pg_ripple.citus_service_pruning=on`.

## Assessment findings covered

From [plans/PLAN_OVERALL_ASSESSMENT_10.md](../plans/PLAN_OVERALL_ASSESSMENT_10.md):

- **HF-3**: Arrow Flight streaming behavior under large exports unverified.

- **MF-7**: Citus HLL accuracy bounds undocumented and untested.
- **MF-8**: Citus SERVICE shard annotation effectiveness not benchmarked.
