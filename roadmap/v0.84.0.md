# v0.84.0 — Assessment 13 Critical/High & Operational Remediation

> **Full technical details:** [v0.84.0-full.md](v0.84.0-full.md)

**Status: Planned** | **Scope: Large**

> v0.84.0 opens the Assessment 13 remediation track. It addresses the highest-priority findings from the post-v0.83.0 code review: HTTP companion readiness improvements, plan-cache correctness, GUC module organization, CI lint hardening, migration-chain test extension, development tooling (justfile recipes), and security annotations (SECURITY DEFINER, audit.toml, strict-compat flag).

---

## What is this?

After the Assessment 12 track closed with v0.83.0, a follow-up code audit (Assessment 13) identified a new set of high-priority operational and correctness issues. v0.84.0 addresses the critical and high-severity findings: service health verification, plan cache correctness, GUC registration hygiene, and CI/security hardening. This is the first of three Assessment 13 remediation releases.

## Headline outcomes

- [ ] **HTTP-READY-01** — `pg_ripple_http` bumped; `COMPATIBLE_EXTENSION_MIN` raised to `"0.84.0"`. The `/health/ready` endpoint performs a real PostgreSQL round-trip (`SELECT 1`) and checks the installed extension version before returning `200 OK`.
- [ ] **PLAN-CACHE-NOPARSE-01** — `cache_key()` in `src/sparql/plan.rs` is refactored to avoid double-parsing the query text. The query is parsed once; the canonical algebra form is used as the cache key.
- [ ] **GUC-MODULE-SPLIT-01** — `src/gucs/registration.rs` is split into per-domain submodules: `src/gucs/storage.rs`, `src/gucs/sparql.rs`, `src/gucs/datalog.rs`, `src/gucs/http.rs`, and `src/gucs/federation.rs`. Each submodule owns the GUC definitions for its domain.
- [ ] **CI-LINT-SQL-01** — `check_no_string_format_in_sql.sh` CI lint step added as a required check. Rejects any `.rs` file that constructs SQL via Rust string formatting (`format!`, `+`, `push_str`) rather than parameterised queries. Closes the class of SQL-injection findings from Assessment 12.
- [ ] **MIGRATION-CHAIN-EXT-01** — `tests/test_migration_chain.sh` extended to assert schema invariants at each migration from `v0.80.0` through `v0.83.0`. Validates column presence, index existence, and GUC registration for each checkpoint.
- [ ] **JUSTFILE-RECIPES-01** — Three new `just` recipes added: `bump-version` (updates `Cargo.toml`, `pg_ripple.control`, and `CHANGELOG.md` in sync), `regen-sbom` (re-runs `cargo cyclonedx` and commits `sbom.json`), and `regen-openapi` (regenerates `docs/openapi/pg_ripple_http.yaml` and validates it).
- [ ] **SECDEF-ANNOT-01** — Every `SECURITY DEFINER` function in the SQL schema annotated with an inline `-- SECURITY-JUSTIFY: <reason>` comment explaining why elevated privileges are required. CI step added to `check_security_definer.sh` to ensure no new `SECURITY DEFINER` is introduced without a justification comment.
- [ ] **AUDIT-TOML-01** — `audit.toml` updated with expiry dates for all `ignore` entries. The `serde_cbor` advisory documented with rationale (evaluation scheduled, ciborium replacement tracked). `PG_RIPPLE_HTTP_STRICT_COMPAT` env var added to `pg_ripple_http` to enable fail-closed extension compatibility check in production deployments.
