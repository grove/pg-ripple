# v0.70.0 - Assessment 10 Critical Remediation

> **Full technical details:** [v0.70.0-full.md](v0.70.0-full.md)

**Status: Released ✅** | **Scope: Large**

> v0.70.0 closes the four Critical and three High findings raised in Overall Assessment 10. Every item has a specific file-and-line root cause and a concrete, bounded fix.

---

## What is this?

Assessment 10 found that the v0.67.0 mutation journal — while correctly wired for single-triple API calls — does not cover bulk load (`load_turtle`, `load_ntriples`, `load_rdfxml`, etc.) and fires the full CONSTRUCT writeback pipeline once per individual triple instead of once per statement. It also found that three evidence-path references in `feature_status()` point to files that do not exist, that the SHACL-SPARQL docs overclaim for the third consecutive assessment, that the README is two releases stale, and that role names in RLS DDL are interpolated without quoting.

v0.70.0 fixes all of these and adds the missing regression tests.

## Headline outcomes

- Bulk load triggers CONSTRUCT writeback rules automatically after every `load_*` call.
- SPARQL Update fires the writeback pipeline once per statement (O(rules)), not once per triple (O(quads × rules)).
- The `validate-feature-status` CI job fails closed when a cited evidence file is missing.
- SHACL docs correctly distinguish between `sh:TripleRule` (supported) and `sh:SPARQLRule` (warning-only deferred to v1.0.0).
- README updated to v0.69.0; version check wired into `just assess-release`.
- Role names in RLS DDL quoted with `quote_ident`; SQL injection test added.
- SBOM regeneration confirmed as a fail-closed step in release CI.
- Legacy `.sh` gate scripts removed; `justfile` calls `.py` versions exclusively.
- `v067_features.sql`, `v069_features.sql`, and `recover_interrupted_promotions` regression test added.
- `roadmap/v0.67.0.md` status corrected to Released ✅.
