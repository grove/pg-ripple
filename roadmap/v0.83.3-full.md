## Background

v0.83.3 is the final Assessment 13 remediation release.  It picks up all findings
not addressed in v0.83.1 (critical/high/operational) or v0.83.2 (correctness/
performance/code-quality).  After this release all 82 A13 findings are resolved.

---

## Test Coverage

### T13-02 — `sparql_roundtrip` Proptest Reference Comparison

The existing proptest suite generates random SPARQL fragments and checks that they
parse without panicking.  It does not compare pg_ripple's *results* against a
reference evaluator.

**Fix**: Extend the proptest harness to evaluate each generated query against the
`spargebra` in-memory evaluator on a small synthetic dataset and assert that the
result multisets are equal.  Start with `SELECT` queries over at most 20 triples.

---

### T13-03 — CONSTRUCT + SHACL-SPARQL Fuzz Targets

Assessment 13 noted that the CONSTRUCT writeback and SHACL-SPARQL evaluation paths
have no fuzz targets despite being complex stateful code.

**Fix**: Add two fuzz targets in `fuzz/fuzz_targets/`:
- `fuzz_construct_writeback.rs` — drives `register_construct_rule()` +
  `run_construct_rules()` with arbitrary Turtle input.
- `fuzz_shacl_sparql.rs` — drives `validate_shacl()` with arbitrary SHACL shapes
  graphs + data graphs.

Both targets are added to the scheduled weekly fuzz CI job.

---

### T13-04 — Conformance Trends CSV Artifact

CI runs each conformance suite but discards the raw counts after the run.  Operators
cannot track regression trends over time.

**Fix**: After each CI run, append a line to `conformance_trends.csv`:
```
date,version,sparql11_pass,sparql11_total,jena_pass,jena_total,watdiv_pass,watdiv_total,lubm_pass,lubm_total
```
Commit the file as a CI artifact and, if running on `main`, commit it back to the
repository so the history is preserved.

---

### T13-05 — `pg_extern` Coverage Gap Audit

Assessment 12 added tests for 13 missing `pg_extern` functions.  Assessment 13 found
that the audit methodology only covered functions reachable from `lib.rs` exports.
Internal helpers exported for testing are not covered.

**Fix**: Run `grep -r '#\[pg_extern\]' src/` and diff against `pg_regress_results/`
test files.  Add regression tests for any function with no corresponding test case.

---

### T13-06 — Benchmark Regression CI Gate

`benchmarks/` contains pgbench scripts but there is no automated gate that fails CI
when throughput regresses.

**Fix**: Add a `benchmark-regression` CI job that:
1. Loads `merge_throughput_baselines.json`.
2. Runs `benchmarks/merge_throughput.sql` for 30 seconds.
3. Compares measured TPS against the baseline; fails if the result is > 10 % below.
4. On `main`, updates the baseline file if the result is a new high-water mark.

---

### T13-07 — CDC Slot Cleanup Crash-Recovery Test

The CDC slot cleanup background worker (`src/storage/cdc.rs`) has no test that
simulates a crash mid-cleanup and verifies that the slot is correctly removed or
marked for retry on the next startup.

**Fix**: Add a `pg_test` that spawns the cleanup worker, uses `pgrx::pg_sys::elog`
to simulate a panic during the cleanup loop, and verifies that the replication slot
is either absent or marked with `active = false` after the extension reinitialises.

## API Polish

### A13-01 — `load_jsonld` Removal Schedule

`load_jsonld()` was deprecated as `json_ld_load()` in v0.53.0 but the old name
still works and the deprecation note in CHANGELOG does not specify a removal target
version.

**Fix**: Add a `BREAKING CHANGE` entry in `CHANGELOG.md` stating that `load_jsonld`
will be removed in v1.0.0, and add a `deprecated` SQL comment to the function.

---

### A13-02 — OpenAPI YAML Committed to Repository

`pg_ripple_http` has an OpenAPI spec generated by `utoipa` but the compiled YAML is
not committed to the repository.  Contributors cannot see what the API looks like
without building the binary.

**Fix**: Add a `just regen-openapi` step (created in v0.83.1) to the CI pipeline
that regenerates `docs/openapi/openapi.yaml` and fails if the result differs from
the committed version.

---

### A13-03 — PT-Code Error Registry

Assessment 13 observed that `PT-xxx` error codes appear in log output and error
messages but are not documented anywhere.

**Fix**: Create `docs/src/reference/error-codes.md` with a table of every `PT-xxx`
code, a description, and the source file/line where it is emitted.  Add a CI step
that greps for `PT-` prefixes in `src/` and fails if any code is not in the
registry.

---

### A13-04 — Deprecated GUCs Documentation

Several GUCs were renamed or replaced between v0.60 and v0.83 but the old names
still work via compatibility shims.  There is no consolidated list of what was
deprecated and what the replacement is.

**Fix**: Add a `## Deprecated GUCs` section to `docs/src/reference/configuration.md`
listing each deprecated GUC, its replacement, and the version it will be removed in.

---

### A13-06 — SPARQL Error Code Standardisation

HTTP responses from `pg_ripple_http` for SPARQL parse errors return HTTP 400 but
the JSON body format differs from the format returned directly by
`sparql_query()` when called via SPI.  Clients must handle two different schemas.

**Fix**: Standardise on the JSON error envelope introduced in v0.83.0
(`AUTH-RESP-FMT-01`): `{ "error": { "code": "PT-xxx", "message": "…" } }` for all
error responses in both the HTTP companion and the extension's `sparql_query()`
SRF function.

## Supply Chain

### DS13-01 — Dependency Freshness Upgrades

Three major-version upgrades are overdue:
- `parquet` 53 → 60
- `ureq` 2 → 3
- `arrow` 53 → 56

**Fix**: Bump each crate; run `cargo test` + `cargo pgrx test pg18` to verify.
Regenerate `sbom.json`.

---

### DS13-03 — SBOM CI Gate

`sbom.json` is regenerated manually.  It has drifted from the actual dependency
tree.

**Fix**: Add a CI step that runs `cargo cyclonedx --format json > sbom.json` and
fails if the result differs from the committed file.

---

### DS13-04 — `rust-toolchain.toml` Update

`rust-toolchain.toml` pins to a specific stable channel from early 2026.  The CI
matrix now tests against a newer toolchain but the toolchain file is not updated.

**Fix**: Update `rust-toolchain.toml` to the current stable channel.

---

### DS13-05 / RR-01 — `tokio-stream` Decision

`pg_ripple_http/Cargo.toml` lists `tokio-stream` as a dependency but Assessment 13
found that the streaming SPARQL cursor implementation is a placeholder (returning a
`Body::from` of pre-collected bytes rather than a true stream).

**Fix**: Either:
- Implement a true chunked-transfer streaming cursor (RR-01), or
- Remove `tokio-stream` from `Cargo.toml` and document the chunked-transfer path
  as a future work item.

The decision must be made and committed; the ambiguity is the finding.

## Observability

### O13-02 — Federation / Dict Cache Metrics

Two important caches have no Prometheus metrics:
- The federation endpoint capability cache (miss = remote HTTP call).
- The in-process dictionary LRU cache (miss = SPI round-trip).

**Fix**: Register four counters:
- `pg_ripple_federation_cache_hits_total`
- `pg_ripple_federation_cache_misses_total`
- `pg_ripple_dict_lru_hits_total`
- `pg_ripple_dict_lru_misses_total`

---

### O13-03 — EXPLAIN Post-Optimiser Algebra

`explain_sparql()` returns the pre-optimiser spargebra algebra.  Operators cannot
see what the optimiser did.

**Fix**: Add an `algebra_after_opt` field to the EXPLAIN JSON output containing the
algebra after `sparopt` optimisation.

---

### O13-04 — JSON Log Output

`pg_ripple_http` writes logs in human-readable format.  Structured log aggregators
(Loki, Datadog) require JSON.

**Fix**: When `PG_RIPPLE_LOG_FORMAT=json` is set, emit logs as
`{"timestamp":"…","level":"…","message":"…","fields":{…}}` using `tracing-json`.

---

### O13-05 — Graceful Shutdown

The axum HTTP server does not register a `SIGTERM` handler.  Kubernetes sends
`SIGTERM` before `SIGKILL`; without a handler, in-flight requests are aborted.

**Fix**: Register a `tokio::signal::unix::signal(SignalKind::terminate())` handler
that calls `axum_server::Handle::graceful_shutdown` with a 30-second timeout.

## Standards Conformance

### SC13-01 — `sparql_strict` GUC Regression Test

`pg_ripple.sparql_strict` GUC is implemented but has no dedicated regression tests
asserting that each filter case it covers (type errors, unbound variables in
arithmetic) raises an error rather than silently returning `false`.

**Fix**: Add a regression test for each documented strict-mode filter case.

---

### SC13-02 — `SERVICE SILENT` Extended Coverage

`SERVICE SILENT` currently swallows connection refused errors.  It does not swallow:
TLS handshake failures, responses > `federation_response_cap`, HTTP 3xx redirects.

**Fix**: Extend the SILENT semantics to all error categories listed in the SPARQL 1.1
Federation spec.  Add regression tests for each.

---

### SC13-03 — GeoSPARQL Function Inventory Documentation

`docs/src/features/geosparql.md` does not list which `geof:` functions are
implemented vs stub vs planned.

**Fix**: Add a `## Function inventory` table with status column.

---

### SC13-04 — DESCRIBE Algorithm Documentation + GUC

The `DESCRIBE` query form uses Concise Bounded Description (CBD) by default.
Symmetric CBD (sCBD) is supported but the choice is not documented and there is
no GUC to control it.

**Fix**: Document the algorithm in `docs/src/reference/sparql-queries.md`.
Add `pg_ripple.describe_form` GUC (`text`, values: `cbd`, `symmetric-cbd`, `scbd`,
default: `cbd`).

---

## HTTP Companion Security

### S13-03 — CORS Audit Log Metric

CORS policy violations (disallowed `Origin` headers) are logged but not counted.
A misconfigured CORS policy can silently block legitimate browser clients.

**Fix**: Register `pg_ripple_http_cors_blocked_total` Prometheus counter; increment
on each rejected preflight.

---

### S13-06 — Bulk-Loader File Path Canonicalization

The bulk-loader endpoint in `pg_ripple_http` accepts a file path parameter and opens
it via the extension's `bulk_load()` SPI call.  The path is not canonicalized before
use, leaving it open to directory traversal sequences like `../`.

**Fix**: Call `std::fs::canonicalize()` on the path before passing it to the SPI
call; return HTTP 400 if the canonical path escapes the configured data directory.

---

### S13-07 — docker-compose.yml Random Password

`docker-compose.yml` hard-codes `POSTGRES_PASSWORD=postgres`.  Developers who
forget to change this before exposing the service will have a well-known credential.

**Fix**: Replace the hard-coded password with
`POSTGRES_PASSWORD: ${POSTGRES_PASSWORD:-$(openssl rand -hex 16)}` and add a `.env`
example file.

---

### S13-08 — Arrow Flight Row Count Info Leak → HTTP 413

The Arrow Flight bulk-export endpoint currently materialises the full result set
before checking row count against the `pg_ripple.arrow_max_rows` GUC.  The count
information is observable via timing even if the 413 is returned.

**Fix**: Estimate row count before materialisation (using `_pg_ripple.predicates`
triple_count) and return HTTP 413 immediately if the estimate exceeds the limit.

---

### S13-09 — Metrics Endpoint Authentication Documentation

The `/metrics` endpoint serving Prometheus metrics is accessible without
authentication by default.  The README does not warn about this.

**Fix**: Add a security note to `pg_ripple_http/README.md` recommending network-level
protection (firewall or `PG_RIPPLE_HTTP_BIND=127.0.0.1`) for the metrics endpoint.

---

### S13-10 — Auth Scheme Documentation

`pg_ripple_http` supports Bearer token and Basic auth but the README only mentions
Bearer.  Basic auth is used in some deployments.

**Fix**: Add a `## Authentication` section to the HTTP companion README documenting
both schemes and the `PG_RIPPLE_HTTP_AUTH_SCHEME` env var.

---

## Performance

### P13-05 — Datalog Rule Streaming in Batches

The `apply_construct_rules()` function loads all registered rules into memory before
evaluation.  For rule sets > 10 000 rules this causes peak memory spikes.

**Fix**: Stream rules in pages of 1 000 using a keyset pagination query on the
catalog table rather than loading all rules at once.

---

### P13-07 — `PathCtx` Field Visibility

`PathCtx` in `src/sparql/path.rs` has public fields that are only used within the
`path` module.  This exposes internal mutable state to the wider `sparql` module.

**Fix**: Change field visibility to `pub(super)` and update the two call sites
outside the module to use accessor methods.

---

## Concurrency

### CC13-03 — CDC LSN Watermark Batching

The CDC watermark update path issues one `UPDATE` per processed event.  Under high
CDC throughput this produces significant write amplification on the watermark table.

**Fix**: Accumulate watermark updates in memory and flush in batches of 100 or every
500 ms, whichever comes first.
