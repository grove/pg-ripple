# v0.9.0 — Serialisation, Export & Interop

**Effort estimate: 3–4 person-weeks**

**Goal:** A knowledge graph is only useful if data can flow in and out in the formats that the rest of the world uses. This release rounds out pg_ripple's format coverage by adding RDF/XML import, Turtle and JSON-LD export, and support for RDF-star in all serialisation paths — making it practical to integrate pg_ripple with virtually any existing RDF toolchain.

---

## RDF/XML Import

**Load knowledge graphs published in the RDF/XML format.**

RDF/XML is the oldest and still widely used serialisation of RDF — it is the native format of OWL ontologies from Protégé, of much of the Linked Open Data cloud, and of many enterprise data exchange standards. This release adds a `load_rdfxml` function that parses RDF/XML documents and loads them through the same dictionary-encode-then-VP-table-insert pipeline used by the Turtle and N-Triples loaders. All existing import patterns (inline content, file path, strict mode) apply equally.

---

## Turtle and JSON-LD Export

**Export any subgraph as Turtle or JSON-LD with a single function call.**

Turtle is the human-readable RDF format preferred for documentation and developer exchange; JSON-LD is the JSON-native RDF format preferred for web APIs and application code. The `export_turtle` and `export_jsonld` functions take a named graph IRI and serialise all its triples into the corresponding format. The JSON-LD output uses the registered prefix table (from `pg_ripple.register_prefix`) to produce compact, readable IRIs rather than full absolute URIs.

---

## SPARQL CONSTRUCT and DESCRIBE in Multiple Formats

**CONSTRUCT and DESCRIBE queries can return their graph in any supported format.**

The CONSTRUCT and DESCRIBE query forms (introduced in v0.5.1) previously returned results as an internal table. This release wires format selection into those query forms: you can ask for the result graph as Turtle, JSON-LD, or N-Triples by passing a format parameter. This makes it straightforward to use pg_ripple as a SPARQL endpoint that returns different formats to different clients based on their Accept header — the groundwork for the SPARQL Protocol HTTP service coming in v0.15.0.

---

## RDF-star Serialisation

**Quoted triples are correctly serialised in all export formats.**

RDF-star extended the N-Triples and Turtle serialisation formats with `<< … >>` notation for quoted triples. This release ensures that any triple where the subject or object is itself a quoted triple is serialised correctly in N-Triples-star and Turtle-star formats on export, and parsed correctly on import. Round-tripping annotated facts through any export/import cycle now preserves their full structure.
