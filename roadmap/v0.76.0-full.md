## Deliverables

### UNWRAP-AUDIT-01 — Audit and fix `unwrap()`/`expect()` outside `#[cfg(test)]` (MF-H)

**Problem**: `src/dictionary/inline.rs` has 13 `unwrap()` calls, `src/flight.rs` has 7, `src/lib.rs` has 5, `src/datalog/builtins.rs` has 4, `src/datalog/stratify.rs` has 3 — all outside test code. Each is a potential backend-crashing panic.

**Deliverable**:
1. For each call: determine if invariant-true (document with `// INVARIANT:` comment) or value-returning (convert to `?` propagation or `pgrx::error!()`).
2. Target: zero uncategorized `unwrap()` in production source files.

**Affected files**: `src/dictionary/inline.rs`, `src/flight.rs`, `src/lib.rs`, `src/datalog/builtins.rs`, `src/datalog/stratify.rs`.

### ROADMAP-VALIDATE-01 — Roadmap status validation script (MF-K)

### RLS-ERROR-01 — Surface `apply_rls_to_vp_table` policy errors (MF-M)

**Problem**: `apply_rls_to_vp_table` at `src/security_api.rs:78-81` and L143 swallows policy creation errors via `let _ =`. Promotion-time RLS failures are invisible.

**Deliverable**:
1. Convert `let _ = ...` to `if let Err(e) = ... { pgrx::warning!("failed to create RLS policy on table {}: {}", oid, e) }`.
2. Add a regression test that triggers a policy error and verifies the warning is emitted.

**Affected files**: `src/security_api.rs`.

### ROLE-DOC-01 — Document ASCII-only role restriction (MF-N)

**Problem**: `is_safe_role_name` rejects roles starting with non-ASCII characters (PG allows `[A-Za-z_\u00A0-\uFFFF]`). The restriction is defensible but undocumented.

**Deliverable**:
1. Update PT711 error message to explicitly state the allowed character set.
2. Add a note in the security reference docs.

**Affected files**: `src/security_api.rs`, `docs/src/operations/security.md` or equivalent.
