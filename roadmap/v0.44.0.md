# v0.44.0 — LUBM Conformance Suite

**Effort estimate: 4–6 person-weeks**

**Goal:** The Lehigh University Benchmark (LUBM) is the standard test suite for OWL reasoning over linked data. Its 14 canonical queries test real-world scenarios drawn from a university domain — faculty, students, courses, departments — and require correct OWL RL inference (subclass hierarchies, property chains, domain/range axioms). All 14 must pass with exact reference cardinalities. This release makes LUBM a required CI check.

---

## LUBM Data Generator Integration

**Synthetic university linked data is generated at controlled scale for repeatable tests.**

The LUBM data generator produces OWL-encoded university data at a configurable scale: `univ=1` generates roughly 100,000 triples representing one university with its faculties, departments, courses, staff, and students. The CI test runs at `univ=1` for speed; the performance baseline test runs at `univ=10` (1 million triples) to verify that query time scales acceptably. Data is loaded once per CI session and cached, so the 14 queries re-use the same loaded dataset.

---

## The 14 Canonical LUBM Queries

**Each query has a published reference answer that pg_ripple must match exactly.**

The LUBM benchmark defines 14 carefully crafted SPARQL queries ranging from simple class membership lookups (Q1: list all students of Professor X) to complex multi-hop inference chains (Q9: find all students who take courses taught by faculty in their own department). The reference cardinalities — the exact number of expected result rows at `univ=1` — are published and stable. For pg_ripple to pass, Datalog inference must correctly materialise all subclass, subproperty, domain, and range entailments before the queries run.

---

## OWL RL Rule Set Validation

**LUBM proves that the built-in OWL RL rule set is correct and complete.**

Passing all 14 LUBM queries is a practical proof that the OWL RL rule set (see v0.10.0) correctly handles the most common entailment patterns. Query 1 tests `rdfs:subClassOf` transitivity. Queries 2, 6, and 7 test `rdf:type` inference through property domains and ranges. Query 9 tests multi-hop property chain traversal. If any OWL RL rule is missing or wrong, at least one LUBM query will fail. Passing all 14 is a necessary (though not sufficient) condition for OWL RL compliance.

---

## Datalog Validation Sub-Suite

**A second test battery verifies the Datalog engine itself, independent of SPARQL.**

Beyond the 14 SPARQL-level LUBM queries, a Datalog-level sub-suite verifies that inference iteration counts, inferred triple counts, and materialization performance all match reference baselines. This catches regressions in the semi-naive evaluation algorithm (v0.24.0), the DRed retraction algorithm (v0.34.0), and the parallel stratum evaluator (v0.35.0) that might not surface in SPARQL-level tests.
