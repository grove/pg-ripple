# v0.4.0 — RDF-star / Statement Identifiers

**Effort estimate: 3–4 person-weeks**

**Goal:** Knowledge graphs often need to say things about their own facts: "Alice knows Bob, according to a 2024 survey", or "the temperature reading was 21°C, measured with 95% confidence". RDF-star provides the standard way to do this — quoting one triple as the subject or object of another — and this release adds full support for storing and querying such annotated facts.

---

## Triples About Triples

**You can now annotate any fact with metadata about that fact.**

Standard RDF can only express simple "subject predicate object" facts. RDF-star extends this by allowing a triple itself to be used as the subject or object of another triple — a "quoted triple". For example, the fact that Alice knows Bob can be annotated with a confidence score, a timestamp, or a source identifier, without creating a separate intermediate node. This is the standard approach for provenance, temporal annotations, and trust metadata in modern knowledge graphs, and it maps naturally to property graphs (LPG) where edges can have their own properties.

---

## Dictionary Encoding for Quoted Triples

**Quoted triples are encoded in the dictionary just like any other term.**

Under the hood, a quoted triple is treated as a special kind of term in the dictionary. The three components of the inner triple (subject, predicate, object) are stored in extra columns on the dictionary row (`qt_s`, `qt_p`, `qt_o`), and the whole quoted triple gets a single integer ID just like an IRI or literal. This means quoted triples participate fully in VP table joins with no special-casing in the query engine — integer joins all the way down.

---

## N-Triples-star Parsing and Serialisation

**Load and export RDF-star data in the N-Triples-star format.**

The N-Triples-star format uses `<< subject predicate object >>` angle-bracket notation to write quoted triples inline in a document. This release adds a parser that recognises this syntax during bulk loading, and an exporter that serialises quoted triples back into the same notation. This makes it straightforward to bring in existing RDF-star datasets and verify that annotated facts survive a round-trip.

---

## SPARQL-star Query Support

**SPARQL queries can now match patterns involving quoted triples.**

The SPARQL query engine is extended to handle ground (fully bound) quoted triple patterns. You can write a SPARQL SELECT that finds all triples annotated with a specific source, or all facts whose confidence exceeds a threshold. The translation to SQL works the same way as for ordinary triple patterns: the quoted triple is encoded to an integer at translation time and matched via an integer equality condition on the VP table.

---

## `encode_triple` and `decode_triple`

**Helper functions let you look up and inspect quoted triple IDs directly.**

The `encode_triple` function accepts three term IRIs and returns the dictionary ID for the corresponding quoted triple, creating it if it does not yet exist. The `decode_triple` function does the reverse — given an ID, it returns the three component terms. These are low-level utilities primarily useful for testing and for tooling that needs to work with the dictionary directly.
