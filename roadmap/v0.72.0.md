# v0.72.0 - Architecture, Protocol Hardening, and Conformance Gates

> **Full technical details:** [v0.72.0-full.md](v0.72.0-full.md)

**Status: Released** | **Scope: Large**

> v0.72.0 addresses the remaining Medium findings from Assessment 10: mutation journal transaction safety under `SAVEPOINT`/rollback, plan cache isolation documentation, continued module split of the largest remaining files, property-based and fuzz testing gaps, conformance suite promotion to required CI, and Arrow Flight replay protection.

---

## What is this?

With Critical and High findings closed by v0.70.0 and v0.71.0, v0.72.0 works through the Medium tier. The most technically significant item is making the mutation journal safe under PostgreSQL subtransaction rollback by registering a `SubXactCallback` that clears accumulated journal entries on abort. A secondary structural item continues the v0.69.0 module split for the two largest remaining files.

## Headline outcomes

- Mutation journal registers `RegisterSubXactCallback` and `RegisterXactCallback`; rolled-back subtransactions no longer produce phantom CONSTRUCT writeback entries.
- Plan cache contract documented; regression test proves same query under two roles with different RLS grants returns identical correct results.
- `src/lib.rs`, `src/storage/mod.rs`, and `pg_ripple_http/src/routing.rs` reduced below 500 lines each via further module extraction.
- Property-based `proptest` suite for `ConstructTemplate` / `apply_construct_template` added.
- Dedicated `fuzz/fuzz_targets/sparql_update.rs` fuzz target exercising SPARQL Update grammar.
- W3C SPARQL 1.1 conformance smoke subset and BSBM regression gate promoted to required (blocking) CI.
- `benchmarks/merge_throughput_history.csv` backfilled for v0.67.0–v0.69.0; weekly trend workflow confirmed to commit rows.
- Arrow Flight ticket nonce-replay protection via server-side seen-nonce LRU cache.
- Datalog/CWB interaction tested: Datalog-derived triples confirmed to trigger downstream CONSTRUCT rules.
- Streaming observability: extension `streaming_metrics()` exposed via a `/metrics/extension` route in `pg_ripple_http`.
- GUC cross-reference docs: `docs/src/reference/guc-reference.md` covers `arrow_batch_size`, `export_batch_size`, and `vp_promotion_batch_size`.
- Citus URL parsing edge-case tests for IPv6, IDN, port-only, and malformed URLs.
- New `pg_ripple.export_jsonld_node()` SQL function: wraps `export_jsonld_framed()` with subject-IRI lookup, `@graph` unwrap, and recursive keyword strip, eliminating the three-step boilerplate from CDC bridge triggers.
- Fix object-form context entries in the inbound JSON-LD parser: enables `@container: @list` (ordered arrays via `rdf:List`), `@type` (datatype hints for typed literals), `@type: @id` (IRI-valued fields), and `@language` (language-tagged strings) — four previously-broken JSON round-trip cases unblocked by one fix. Outbound framing embedder updated to honour the same context features.
- Documented JSON round-trip corner cases in the spec: empty arrays, multi-graph ingest, IRI-component validation, numeric precision, null-vs-absent, and duplicate keys are tracked as separate backlog items or documented as inherent JSON ↔ RDF impedance mismatches.
- Targeted ingest fixes for three small JSON round-trip bugs: i64 overflow at ingest no longer silently loses precision (RT-FIX-04B); JSON `5.0` is now stored as `xsd:decimal` rather than collapsing to `xsd:integer` (RT-FIX-06); invalid IRI characters in JSON keys raise a descriptive error instead of producing malformed IRIs (RT-FIX-07).
- Empty-object regression test added (RT-TEST-02).
- Property-test generator for heterogeneous JSON arrays added to `PROPTEST-01` (RT-10).

## Assessment findings covered

From [plans/PLAN_OVERALL_ASSESSMENT_10.md](../plans/PLAN_OVERALL_ASSESSMENT_10.md):

- **HF-6**: Plan cache key omits graph context and security context.
- **HF-7**: Mutation journal not safe under `SAVEPOINT`/`ROLLBACK TO SAVEPOINT`.
- **MF-5**: `merge_throughput_history.csv` has only one row; trend gate cannot fire.
- **MF-10**: `src/lib.rs` (~2,300) and `src/storage/mod.rs` (~2,100) still too large.
- **MF-11**: No proptest for `ConstructTemplate` / `apply_construct_template`.
- **MF-12**: Fuzz corpus may not exercise SPARQL Update grammar.
- **MF-13**: Streaming observability counters not exposed via `/metrics`.
- **MF-14**: Three batch-size GUCs undocumented relative to each other.
- **MF-17**: Datalog/CWB interaction undocumented and untested.
- **MF-18**: `is_citus_worker_endpoint()` URL parsing untested for edge cases.
- **MF-19**: Arrow Flight ticket has no replay protection beyond expiry.
- **UX-01**: CDC bridge triggers require three-step boilerplate (subject decode, `@graph` unwrap, keyword strip) when using `export_jsonld_framed()`.
