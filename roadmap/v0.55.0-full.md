## v0.55.0 — Security Hardening & v1.0.0 Unblocking

**Theme**: Close every v1.0.0-blocking finding from PLAN_OVERALL_ASSESSMENT_6: federation SSRF, HTAP merge race, error-catalog drift, mid-merge crash test, concurrent-write stress test, and a suite of code-quality, standards, and ecosystem quick wins.

> **In plain language:** A SPARQL `SERVICE <…>` query can currently cause the PostgreSQL server to make arbitrary outbound HTTP requests — including to cloud metadata endpoints like `169.254.169.254`. This release adds a network-policy GUC that blocks unsafe destinations by default. The HTAP merge worker's brief "relation does not exist" window is eliminated. The error-catalog is reconciled. The shacl/mod.rs god-module is split. Dead-code markers in DRed, compiler, and Datalog modules are resolved. Named-graph row-level security and GDPR erasure tooling arrive. Ecosystem quick wins deliver VoID, Service Description, Jupyter kernel, LangChain integration, Kafka CDC sink, read-replica routing, and materialized SPARQL views.
>
> **Effort estimate: 6–8 person-weeks**
