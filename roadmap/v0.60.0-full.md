# v0.60.0 — Production Hardening Sprint — Full Details

**Theme:** Close every v1.0.0 blocker surfaced in PLAN_OVERALL_ASSESSMENT_7.
**Scope:** Large  
**Prerequisite:** v0.59.0

---

## Why this release

PLAN_OVERALL_ASSESSMENT_7 (v0.59.0 baseline) raised the overall maturity score
to 4.91/5.0 — but two architectural defects and several supply-chain gaps remain
before v1.0.0 can be tagged with confidence. v0.60.0 exists to close those gaps
with no new feature scope: every item is either a correctness fix, a security
gate, a test net, or a quick-win feature completion.

---

## Deliverables

### Storage / Correctness

- **[F7-1] HTAP merge cutover atomic swap** — Replace the
  `DROP TABLE … CASCADE → RENAME → CREATE OR REPLACE VIEW` sequence in
  `src/storage/merge.rs` with an `ACCESS EXCLUSIVE`-locked rename-swap that
  never leaves the underlying relation non-existent. The view definition stays
  stable; only the table backing it is replaced atomically.
- **[J7-1] HTAP cutover chaos test** — New shell test
  `tests/concurrent/merge_cutover_chaos.sh` that runs continuous
  `pgbench SELECT` traffic against the view while the merge worker churns;
  asserts zero `relation does not exist` errors over a 60-second window.
- **[F7-2] Rare-predicate promotion concurrency test** — New
  `tests/concurrent/promotion_race.sh` driving two parallel sessions across
  `vp_promotion_threshold` and asserting exactly one VP table is created.
- **[F7-4] Merge-throughput trend artifact** — Append a
  `(release_tag, p50_tps, p95_tps)` row to
  `benchmarks/merge_throughput_history.csv` during every release workflow run.

### Security / CI supply-chain

- **[H7-1] GitHub Actions SHA pinning** — Pin every external action in
  `.github/workflows/*.yml` to an immutable SHA digest; configure Dependabot
  `package-ecosystem: github-actions` to keep them current.
- **[H7-2] SECURITY DEFINER CI lint** — Gate
  `scripts/check_no_security_definer.sh` as a required CI step; allowlist the
  two known DDL-trigger functions explicitly.
- **[H7-3] Security doc fix** — Update `docs/src/reference/security.md:198`
  to clarify that only the DDL-guard event trigger uses `SECURITY DEFINER`.
- **[N7-1/N7-2] Rust toolchain version pin** — Add/update `rust-toolchain.toml`
  to pin the exact stable channel version; update `dtolnay/rust-toolchain` to
  reference a SHA-pinned action.
- **[N7-4] Docker CVE scan** — Add `aquasecurity/trivy-action` (SHA-pinned) to
  the release workflow targeting the `Dockerfile.batteries` image.

### Fuzz coverage

- **[A7-1] New fuzz targets** — Add three fuzz harnesses:
  - `fuzz/fuzz_targets/geosparql_wkt.rs` — exercises the GeoSPARQL WKT geometry
    parser via PostGIS `ST_AsWKT` round-trip.
  - `fuzz/fuzz_targets/r2rml_mapping.rs` — exercises the R2RML mapping document
    parser in `src/r2rml.rs`.
  - `fuzz/fuzz_targets/llm_prompt_builder.rs` — exercises the prompt sanitizer
    in `src/llm/mod.rs`; asserts no prompt-injection marker survives.

### Rust code quality

- **[A7-2] Remove false-positive `#[allow(dead_code)]`** on
  `execute_with_savepoint` in `src/datalog/parallel.rs:340` — the function is
  now actively called from `coordinator.rs`.
- **[A7-4] `clippy --deny warnings` CI gate** — Add
  `cargo clippy --workspace --all-targets --all-features -- -D warnings` as a
  required lint step in the CI workflow.

### SPARQL engine

- **[B7-3] `geof:distance`** — Add `geof:distance(?a, ?b, <units>)` to the
  GeoSPARQL function dispatch table in `src/sparql/expr.rs`; map to PostGIS
  `ST_Distance(g1, g2)`.
- **[B7-4] `SERVICE SILENT` + circuit-breaker interaction** — Document and add
  a pg_regress test asserting that `SERVICE SILENT` correctly swallows PT605
  (circuit-breaker-open) and returns the empty solution sequence per the SPARQL
  1.1 spec.

### HTTP companion

- **[H7-5] `/ready` endpoint** — Add a `/ready` route to `pg_ripple_http` that
  returns 503 until the first successful PostgreSQL connection check; `/health`
  remains the liveness probe, `/ready` becomes the Kubernetes readiness probe.

### Documentation & developer experience

- **[K7-1] Architecture diagram refresh** — Update the mermaid diagrams in
  `docs/src/reference/architecture.md` to include `src/citus.rs`,
  `src/tenant.rs`, `src/kge.rs`, `src/temporal.rs`, `src/sparql/sparqldl.rs`,
  `src/sparql/ql_rewrite.rs`.
- **[K7-2] pg_trickle dependency matrix** — Add a feature-matrix table to
  `README.md` listing which pg_ripple features require pg_trickle vs. ship
  standalone.
- **[K7-3] Citus rebalance example** — Add
  `examples/citus_rebalance_with_trickle.sql` runnable walkthrough.
- **[6.10] LangChain / LlamaIndex tool package** — Publish a `pg_ripple` Python
  tool class for LangChain and LlamaIndex in a separate `clients/python/`
  directory; include the `pg_ripple.repair_sparql()` hook.

### Test completeness

- **[6.14] `pg_dump` round-trip CI test** — Add `tests/integration/dump_restore.sh`
  that populates a pg_ripple instance, runs `pg_dump`, restores to a fresh
  instance, and asserts that all triple counts, inference results, and SHACL
  report hashes match.
