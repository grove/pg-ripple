# v0.7.0 — SHACL Validation (Core)

**Effort estimate: 4–6 person-weeks**

**Goal:** Raw knowledge graphs tend to drift: missing required fields, wrong data types, values outside allowed ranges. This release introduces SHACL (Shapes Constraint Language) — the W3C standard for expressing and enforcing data quality rules on RDF graphs. You write your rules once as a Turtle document, load them into pg_ripple, and the engine enforces them automatically on every insert.

---

## Loading SHACL Shapes

**Define data quality rules as a Turtle document and load them with a single function call.**

SHACL shapes are themselves RDF data — you write them in Turtle format, describing which entities must have which properties, what types values must be, how many values are allowed, and so on. The `load_shapes` function parses that document and stores the compiled constraint logic ready for enforcement. This means your data contracts live in the same version-controlled, portable format as your data.

---

## Synchronous Validation on Insert

**Triples that violate a shape are rejected immediately, before any data is stored.**

When the `pg_ripple.shacl_mode` configuration is set to `sync`, every call to `insert_triple` or the SPARQL Update operations triggers a constraint check before committing. If the new triple would violate any loaded shape, the insert is rejected with a structured error message identifying which constraint failed and which entity was the focus of the violation. This gives the same strong guarantee as a database NOT NULL or CHECK constraint: invalid data simply cannot enter the store.

---

## `validate()` Function

**Run a full validation scan across all loaded shapes and get a complete violation report.**

The `validate()` function scans the entire graph against all loaded shapes and returns a table of every violation found. Each row identifies the focus node (the entity that failed), the violated shape, the specific constraint component, and the offending value. This is useful for auditing existing data (before enabling synchronous enforcement), for batch validation jobs, and for SHACL-aware data pipelines that need a machine-readable quality report.

---

## Explicit Deduplication

**Companion functions let you identify and remove duplicate triples.**

The release also provides deduplication helper functions that scan VP tables for logically identical triples and remove the extras. This is primarily useful after bulk loads that may have inserted the same fact from multiple source files, or after merging data from two knowledge graphs with overlapping content. An optional merge-time deduplication mode can also be enabled so that each merge cycle cleans up duplicates automatically.
