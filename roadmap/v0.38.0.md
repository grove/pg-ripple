## Breaking Up the Monolith

**The central `src/lib.rs` file is split into focused, maintainable modules.**

A single 5,600-line file that contains registration logic, GUC definitions, federation registry code, GraphRAG admin functions, stats collection, and more is a maintenance problem: every change touches the same file, creating merge conflicts and making it hard to find code. This release extracts each functional area into its own module. The target is a `lib.rs` under 1,500 lines covering only the entry points and wiring, with all substantive logic in separate files. No public SQL API changes: every existing `pg_ripple.*` function still works the same way.

## `PredicateCatalog` Abstraction

**A clean interface between the query engine and the VP table storage layer.**

The SPARQL translator and Datalog compiler both need to look up which physical table stores a given predicate. Previously both made direct SPI calls to the `_pg_ripple.predicates` table on every predicate they encountered. This release introduces a `PredicateCatalog` trait and a backend-local in-memory cache that holds the lookup results for the current session. The cache is invalidated by a PostgreSQL syscache callback when the predicates table changes. A 10-predicate BGP that previously made 10 catalog SPI calls now makes one.

## SHACL Hints Wired into SPARQL Planner

**Shape constraints now actually influence query plan choices.**

SHACL shapes that declare `sh:maxCount 1` (at most one value per subject) and `sh:minCount 1` (always at least one value) have always been promised to influence SPARQL query plans â€” `maxCount 1` should suppress DISTINCT, `minCount 1` should upgrade OPTIONAL to INNER JOIN. This was documented but not actually implemented. This release verifies the hints are applied and adds a test that uses `explain_sparql()` to confirm the plan changes.

---

## Plan Cache Key Normalisation

**Whitespace variants and prefix differences in SPARQL queries share the same cache slot.**

Two SPARQL queries that ask the same question but differ in whitespace formatting or in how they abbreviate IRIs (full IRI vs. prefix:localname) previously got separate cache entries. The plan cache now keys on the normalised query algebra digest (a compact hash of the internal algebra tree, not the query text), so whitespace-equivalent queries share the same cached plan. This significantly improves cache hit rates for applications that generate SPARQL programmatically.
