# v0.17.0 — JSON-LD Framing

**Effort estimate: 4–6 person-weeks**

**Goal:** Flat lists of triples are not always the most useful format for application code. JSON-LD Framing is a W3C standard that lets you provide a "frame" — a template for the shape of the JSON you want — and get back a nested, structured JSON-LD document that matches that shape. This release implements framing as a first-class pg_ripple feature, with the crucial twist that frames translate directly to SPARQL CONSTRUCT queries.

---

## W3C JSON-LD Framing

**Provide a shape template and get back nested, structured JSON.**

A JSON-LD frame is a JSON-LD document that describes the structure you want in the output: which types of entities to find, which properties to include, and how to nest related entities. For example, you might frame a graph of people and organisations to get back a JSON array where each organisation contains a nested array of its employees, each employee contains their skills, and so on. The framing algorithm processes the flat triple store and reshapes it into this hierarchical structure, which is far more natural to consume in application code.

---

## Frame-to-SPARQL Translation

**Frames are compiled to SPARQL CONSTRUCT queries, not naive graph traversal.**

Rather than loading all triples into memory and applying the framing algorithm in the application, pg_ripple compiles the frame to a SPARQL CONSTRUCT query that touches only the specific VP tables referenced by the frame. A frame asking for Person entities with their name and employer only reads the `rdf:type`, `foaf:name`, and `org:memberOf` VP tables — not the entire graph. For large graphs this difference is dramatic.

---

## `export_jsonld_framed()` Function

**Call a single SQL function to get a framed JSON-LD document.**

The `export_jsonld_framed(graph_iri TEXT, frame_json TEXT)` function accepts a graph IRI and a JSON-LD frame as text, and returns the framed JSON-LD document as a single JSONB value. The function handles all the steps: translating the frame to a CONSTRUCT query, executing it, applying the W3C JSON-LD embedding algorithm (which handles blank-node cycles), and assembling the nested output.

---

## Optional Live Framing Views

**With pg-trickle installed, framed views can update incrementally.**

If pg-trickle is present, a framed JSON-LD document can be registered as a live view — a table where each row is a JSON-LD fragment for one entity, kept up to date as triples change. This is useful for API backends that need to serve entity-detail views without running a full framing query on every request.
