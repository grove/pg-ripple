## Area 1: Correctness (C13-02 – C13-11)

### C13-02 | MEDIUM | Effort: S
`batch_decode` returns a silent empty string when an ID is missing from the
dictionary, even when `pg_ripple.strict_dictionary = on`.
**Fix:** Return `PT512_DICTIONARY_MISS` error when strict mode is active.

### C13-03 | MEDIUM | Effort: S
Blank nodes inside RDF-star quoted triples do not have a canonical form;
round-trip through encode/decode produces different blank-node labels.
**Fix:** Document limitation; add round-trip regression test; either reject
with PT512 or canonicalise using a deterministic scheme.

### C13-04 | MEDIUM | Effort: S
`execute_drop()` and `execute_clear()` do not flush the mutation journal before
returning. If the caller crashes after the operation but before flush the VP
table is inconsistent.
**Fix:** Either document the flush obligation with a comment at the call site,
or add explicit `record_clear()` + `flush()` inside the functions.

### C13-05 | MEDIUM | Effort: S
Plan cache key is case-sensitive for the `INFERENCE_MODE` header value. Queries
with `inference_mode: NONE` and `inference_mode: none` are cached separately.
**Fix:** Normalise `INFERENCE_MODE` with `trim() + to_lowercase()` before hashing.

### C13-06 | MEDIUM | Effort: S
`GRAPH ?g { ... }` excludes the default graph from `?g` bindings but this
behaviour is not documented. Operators unfamiliar with SPARQL 1.1 §8.3 are
surprised.
**Fix:** Document in `docs/src/reference/sparql-compliance.md`; add regression
test asserting the default-graph exclusion.

### C13-07 | MEDIUM | Effort: S
`batch_decode` warning guard is `id <= 0` but negative IDs should not exist
after the v0.81.0 dict_subxact fix. Only `id == 0` is the sentinel.
**Fix:** Tighten guard to `id == 0`.

### C13-08 | MEDIUM | Effort: S
`encode_token` in `src/datalog/magic.rs` does not distinguish plain strings
from typed literals (`^^<` suffix), so typed literals are encoded as plain
strings and lose their type annotation.
**Fix:** Add detection for `^^<` and route to `encode_typed_literal()`.

### C13-09 | MEDIUM | Effort: S
`parse_nt_triple` does not validate IRI length or require a closing `>`.
Malformed input silently produces corrupt IDs.
**Fix:** Reject IRIs longer than 4 KiB; require trailing `>` for IRI tokens.

### C13-10 | MEDIUM | Effort: S
`xsd:dateTime` values with sub-millisecond precision (e.g. `.123456789Z`) are
silently truncated to 3 decimal places. This is not documented.
**Fix:** Document the 3-decimal-place limit; add a round-trip regression test.

### C13-11 | MEDIUM | Effort: S
`describe_cbd()` has no recursion depth cap. On a cyclic or very deep graph the
CBD traversal runs indefinitely.
**Fix:** Add `pg_ripple.describe_max_depth` GUC (default 16); raise
`PT_DEPTH_LIMIT` error when exceeded.

## Area 2: Performance (P13-02 – P13-08)

### P13-02 | MEDIUM | Effort: M
`encode_inner()` issues one SPI call per term. Bulk loads of N triples issue
3N SPI calls for subject, predicate, object.
**Fix:** `encode_batch(terms: &[Term]) -> Vec<i64>` issues a single CTE
`INSERT ... ON CONFLICT DO NOTHING ... RETURNING id` and returns all IDs.

### P13-03 | MEDIUM | Effort: S
Merge-worker heartbeat log line emitted every cycle (~1 second). Produces
excessive log volume in production.
**Fix:** Throttle to once per 60 seconds; keep the `merge_worker_status` table
update on every cycle.

### P13-04 | MEDIUM | Effort: S
`execute_select()` issues `SET LOCAL pg_ripple.inference_mode = ...` and
`SET LOCAL pg_ripple.graph_ids = ...` as separate SPI calls before each query.
**Fix:** Batch into a single SPI call: `SET LOCAL ...; SET LOCAL ...;`.

### P13-05 | MEDIUM | Effort: S
Datalog rule streaming sends one rule at a time to the SPI engine.
**Fix:** Batch rules in groups of 100 before sending.

### P13-06 | MEDIUM | Effort: M
`partition_into_parallel_groups()` in `src/datalog/parallel.rs` does not
pre-check for cycles in head-group dependency before running the SCC
partitioner. A cycle causes an unbounded recursion.
**Fix:** Add a cycle check on the head-group dependency graph before invoking
SCC.

### P13-07 | MEDIUM | Effort: S
`PathCtx.counter` field is `pub` — callers can mutate it directly, bypassing
alias uniqueness guarantees.
**Fix:** Make field private; add `pub fn next_alias(&mut self) -> String`.

### P13-08 | MEDIUM | Effort: S
Dictionary hot-cache hits and misses are not instrumented with Prometheus
metrics.
**Fix:** Add `dictionary_hot_cache_hits_total` and
`dictionary_hot_cache_misses_total` counters in
`pg_ripple_http/src/metrics.rs`.

## Area 3: Code Quality (Q13-02 – Q13-05)

### Q13-02 | MEDIUM | Effort: M
`src/schema.rs` is 1,939 lines — the second-largest file after
`gucs/registration.rs` (resolved in v0.84.0).
**Fix:** Split into `schema/tables.rs`, `schema/views.rs`, `schema/triggers.rs`,
`schema/rls.rs`.

### Q13-03 | MEDIUM | Effort: M
`src/sparql/federation.rs` is 1,693 lines.
**Fix:** Split into `federation/circuit.rs`, `federation/policy.rs`,
`federation/http.rs`, `federation/decode.rs`.

### Q13-04 | MEDIUM | Effort: M
`src/datalog/compiler.rs` (1,613 lines), `src/storage/ops.rs` (1,547 lines),
and `src/sparql/expr.rs` (1,498 lines) are approaching the 1,800-line danger
zone.
**Fix:** Add a CI lint step that fails when any `src/**/*.rs` file exceeds
1,800 lines unless the file contains `// @allow-large-file: <reason>`.

### Q13-05 | MEDIUM | Effort: S
36 `#[allow(dead_code)]` markers across `src/`. Most are legitimate future API
surfaces but one borderline case at `src/datalog/parallel.rs:51` has a
"not yet" comment.
**Fix:** Audit each marker; wire up or delete.

## Area 4: Concurrency (CC13-01 – CC13-02)

### CC13-01 | MEDIUM | Effort: M
`promote_predicate_impl()` at `src/storage/promote.rs:54–100` acquires an
advisory lock then sets `promotion_status = 'promoting'` BEFORE the atomic
CTE. A SIGKILL between those two operations leaves the catalog in a
`'promoting'` state indefinitely.

`recover_interrupted_promotions()` exists at `src/storage/promote.rs:139` but
it is not confirmed to run on every backend start.

**Fix:** Verify `recover_interrupted_promotions()` is called at backend start.
Add `tests/crash_recovery/promote_sigkill.sh` that:
1. Starts a promotion
2. SIGKILLs the backend between status update and CTE
3. Restarts and asserts the predicate is in the correct final state

### CC13-02 | MEDIUM | Effort: S
Merge fence lock is global advisory lock at `0x5052_5000` (A12 CC-04; not
fixed in v0.80–v0.83). All predicates serialise through one lock even when
merging unrelated predicates concurrently.
**Fix:** Scope the lock per-predicate using `pg_advisory_xact_lock(0x5052_5000
| hash(predicate_id))`; keep the global lock only for Citus rebalance.
