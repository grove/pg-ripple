# v0.81.0 — Assessment 12 Correctness & Concurrency Closure

> **Full technical details:** [v0.81.0-full.md](v0.81.0-full.md)

**Status: Planned** | **Scope: Large**

> v0.81.0 resolves the thirteen correctness findings and seven concurrency findings from Assessment 12. The theme is robustness: fixing non-deterministic SID selection in HTAP merges, promotion TOCTOU races, dictionary SubXact cache invalidation, CDC slot lifecycle, and a cluster of Datalog, blank-node scoping, federation, and SHACL correctness issues. A new `strict_sparql_filters` GUC and `strict_dictionary` GUC are added for forward compatibility.

---

## What is this?

After v0.80.0 clears the v1.0.0 blockers, v0.81.0 addresses Assessment 12's correctness and concurrency findings — the B-priority items that represent data-integrity risks or race conditions that may surface under production load.

## Headline outcomes

- [x] **MERGE-SID-01** — Fix non-deterministic SID selection in HTAP merge by including `ORDER BY i ASC` before `DISTINCT ON` in the merge CTE template.
- [x] **PROMO-ATOMIC-01** — Move `_pg_ripple.predicates` status update inside the atomic CTE that inserts the new VP table, eliminating the promotion TOCTOU window.
- [x] **DICT-RACE-01** — Error (do not panic) when `encode()` receives 0 rows from RETURNING, indicating a hash collision or concurrent dict truncation.
- [x] **SHACL-TXN-01** — Wrap the SHACL shape-store write and DDL hint insertion into a single subtransaction so a constraint failure rolls back both.
- [x] **FED-URL-01** — Normalise federation endpoint URLs to lowercase scheme+host before allowlist comparison.
- [x] **FED-TRUNC-01** — Replace silent truncation of federation JSON results exceeding `federation_result_max_bytes` with a WARNING and partial materialisation.
- [x] **OPT-INNER-01** — Extend the OPTIONAL→INNER JOIN optimisation to multi-predicate BGPs in `src/sparql/plan.rs`.
- [x] **DL-AGG-01** — Guard against aggregation functions in recursive Datalog rule heads; emit a descriptive error rather than silently producing incorrect results.
- [x] **DL-PAR-01** — Detect intra-stratum cycles in the parallel group partition step to prevent non-terminating stratum evaluation.
- [x] **BN-SCOPE-01** — Prefix blank-node variable names with a query-scoped UUID to prevent cross-subquery blank-node aliasing.
- [x] **FED-CACHE-01** — Normalise federation query cache keys (lowercase, whitespace-collapsed) to improve hit rates and prevent cache key splitting on equivalent queries.
- [x] **DICT-STRICT-01** — Add `pg_ripple.strict_dictionary` GUC; when enabled, `decode()` returns an error instead of a fallback string for missing IDs.
- [x] **DICT-SUBXACT-01** — Register a `SubXactCallback` that invalidates the dictionary decode LRU cache on subtransaction abort.
- [x] **CDC-SLOT-01** — Add a background worker that monitors for orphaned CDC replication slots and drops them after a configurable idle timeout.
- [x] **PROMO-LOCK-01** — Replace the single `_pg_ripple.predicates` table-level lock with per-predicate advisory locks during VP table promotion.
- [x] **MERGE-FENCE-01** — Reduce the merge fence `ShareUpdateExclusiveLock` scope to only the BRIN-reindex phase; the main-partition swap phase uses `ExclusiveLock` with a shorter hold.
- [x] **DL-PAR-02** — Correct the Datalog parallel SCC cycle check to use topological order rather than stratum order.
- [x] **CDC-LSN-01** — Add `_pg_ripple.cdc_lsn_watermark(slot_name, last_lsn)` table and update it after each CDC batch commit.
- [x] **PROMO-STUCK-01** — Add `pg_ripple.recover_stuck_promotions()` SQL function that detects and rolls back promotions abandoned mid-flight without requiring a server restart.
- [x] **FILTER-STRICT-01** — Add `pg_ripple.strict_sparql_filters` GUC; when enabled, unknown SPARQL built-in function names produce an error rather than evaluating to `UNDEF`.
- [x] **REPL-UNWRAP-01** — Replace all `.unwrap()` and `.expect()` calls in `src/replication.rs` with `?`-propagated errors or explicit `pg_error!()` macro calls.
- [x] **PGFINI-01** — Add a `_PG_fini` C function that unregisters the SubXact callback, ExecutorEnd hook, and transaction callback on `DROP EXTENSION`. Currently they dangle in PostgreSQL's callback lists and may fire after the extension library is unloaded.
- [x] **PLAN-CACHE-GUC-02** — Extend the plan cache key in `src/sparql/plan_cache.rs` to include `NORMALIZE_IRIS`, `WCOJ_ENABLED`, `WCOJ_MIN_TABLES`, and `INFERENCE_MODE`. Changing any of these GUCs mid-session currently returns stale plans compiled under old values.
- [x] **FEATURE-STATUS-BIDI-01** — Add 12 missing `feature_status()` rows for BIDI (v0.77.0) and BIDIOPS (v0.78.0) features: `bidi_integration`, `bidi_conflict_policy`, `bidi_upsert_mode`, `bidi_diff_mode`, `bidi_linkback`, `bidiops_queue_depth_limits`, `bidiops_pause_resume`, `bidiops_schema_evolution`, `bidiops_per_subscription_auth`, `bidiops_frame_redaction`, `bidiops_audit_trail`, `bidiops_reconciliation`.
- [x] **PRELOAD-WARN-01** — Emit a `WARNING` in `_PG_init` when loaded via `CREATE EXTENSION` (not `shared_preload_libraries`): `"pg_ripple: loaded without shared_preload_libraries; HTAP merge worker, CONSTRUCT writeback, and dictionary cache are disabled"`. Prevents silent misconfiguration.
- [x] **RETRACT-PARAM-01** — Parameterise the flat-VP `DELETE` in `src/construct_rules/retract.rs` to use `Spi::run_with_args()` with `$1`–`$4` bind parameters, consistent with the `vp_rare` retraction path on the same function.
- [x] **SCHEDULER-ERR-01** — Replace `pgrx::error!()` in `src/construct_rules/scheduler.rs` topological sort with a `Result`-returning error path so callers can handle the "internal invariant violated" case gracefully rather than immediately aborting the backend.
- [x] **DRED-FIXPOINT-01** — Fix `src/datalog/dred.rs` re-derivation to run the full Datalog fixpoint (not just one seed pass) when deleted triples have multi-level dependent derivations. Add regression test: 3-hop chain where removing the base triple must retract all derived facts.
- [x] **RETRACT-HTAP-01** — Guard the flat-VP `DELETE` path in `src/construct_rules/retract.rs` against HTAP-state inconsistency: if the row is in `main` (not `delta`), insert a tombstone instead of executing a direct DELETE against the flat table.
- [x] **RAG-SQL-INJECT-02** — Replace manual SQL string escaping in `pg_ripple_http/src/routing/rag_handler.rs` (currently `format!()` + `replace('\''` + `"'"`)`) with parameterized `client.query($1, $2, ...)` calls. The existing comment "parameterized queries prevent SQL injection" is incorrect — the handler uses `format!()`. Eliminates risk when `standard_conforming_strings=off`.
- [x] **PLAN-CACHE-GUC-02** (extended) — In addition to `NORMALIZE_IRIS`, `WCOJ_ENABLED`, `WCOJ_MIN_TABLES`, `INFERENCE_MODE` (already listed), also add `TOPN_PUSHDOWN`, `SPARQL_MAX_ROWS`, `SPARQL_OVERFLOW_ACTION`, `FEDERATION_TIMEOUT`, and `PGVECTOR_ENABLED` to the plan cache key. All five affect query generation or execution limits and are not currently in the cache key.
