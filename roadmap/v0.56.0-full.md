## v0.56.0 — Standards Completeness & Operational Depth

**Theme**: GeoSPARQL 1.1, SPARQL Entailment Regime conformance tests, Apache Arrow/Flight bulk export, dbt adapter, federation circuit breaker, DDL event trigger, SPARQL audit log, whole-file dead-code audit, and removal of the deprecated `property_path_max_depth` GUC.

> **In plain language:** Building on the PostGIS bundling in v0.54.0, this release wires the full GeoSPARQL 1.1 geometry function vocabulary so users can ask "find all hospitals within 5 km of this point" directly in SPARQL. A formal conformance test suite for SPARQL Entailment Regimes verifies that the OWL 2 RL + SPARQL combination behaves correctly per the W3C spec. A native Arrow Flight endpoint lets data engineers stream large graph exports at columnar speed. A dbt adapter makes RDF graph data available to the dbt transformation ecosystem. The federation circuit breaker prevents one failing remote endpoint from degrading the entire query.
>
> **Effort estimate: 5–7 person-weeks**

- [ ] Implement GeoSPARQL 1.1 geometry functions as SPARQL FILTER extensions in `src/sparql/expr.rs`:
  - `geof:distance(geom1, geom2, units)` → `ST_Distance`
  - `geof:within(geom1, geom2)` → `ST_Within`

  - `geo:hasSpatialAccuracy(iri)` → literal lookup

- [ ] Guard all `geof:*` functions with a runtime check for PostGIS availability; raise PT532 with helpful message if PostGIS is absent
- [ ] `docs/src/reference/geosparql.md` updated with function reference and worked examples
